DAY 20 OF 90 — PHASE 2

Web Recon: Directory Busting & Technology Fingerprinting

You know a website's front door exists (Day 8 HTTP). Today you find every side-door, maintenance hatch, backup stairwell, and forgotten window — the hidden paths attackers use to bypass logins and inject malware. Real attackers discovered the British Airways Magecart injection vector this exact way.

THEORY

Web recon is the process of mapping a web application's full attack surface — not just the pages users click, but every file, directory, parameter, and technology behind it. Directory busting (brute-forcing URLs with a wordlist) is the primary technique. Technology fingerprinting identifies what stack the site runs — Apache vs Nginx, PHP vs React, Cloudflare vs no WAF — so you know which vulnerabilities to test.

Gobuster
Directory & DNS brute-forcer
Fastest directory scanner written in Go. Brute-forces URL paths, virtual hosts, and DNS subdomains using a wordlist. Main command: gobuster dir -u target -w wordlist.txt. Returns HTTP status codes — 200 (found), 403 (forbidden but exists), 301 (redirect).
FFuF
Fast web fuzzer
Multipurpose fuzzer — not just directories but parameters, headers, and POST bodies. Replaces FUZZ in any part of the request with wordlist entries. Uses Go routines for extreme speed. ffuf -u http://target/FUZZ -w wordlist.txt
Wappalyzer
Browser extension / API
Identifies ~1500 web technologies from JavaScript frameworks to CDN providers. Runs as a browser extension or CLI via wappalyzer-cli. Passive — just reads response headers and page source. No aggressive requests.
WhatWeb
CLI technology detector
Broadest tech fingerprinting CLI tool. Identifies CMS, JS libraries, server headers, analytics, even country of hosting. whatweb example.com returns 10–30 tech tags instantly. Scriptable for bulk scanning.
Common find: /wp-admin (WordPress admin) Common find: /admin (generic panel) Common find: /.git (exposed repo) Common find: /backup (old backups) Common find: /api (REST endpoints) Common find: /server-status (Apache status) Common find: /.env (credentials file) Common find: /sitemap.xml (crawl map)
Wordlists matter: Kali includes wordlists at /usr/share/wordlists/. The most versatile is dirb/common.txt (4,600 entries) for small targets or directory-list-2.3-medium.txt (220,000 entries) from SecLists for comprehensive scans. Install SecLists with sudo apt install seclists — it's the standard wordlist for pentesting.
// real named incident
2018 — BRITISH AIRWAYS / MAGE CART GROUP — 380,000 PAYMENT CARDS STOLEN
Attackers used directory busting to find an unsecured admin panel on ba.com
In 2018, the Magecart Group (a cluster of threat actors specializing in web-based credit card theft) targeted British Airways' main website, ba.com. Through directory enumeration they discovered a hidden admin panel path — a forgotten staging endpoint that was still accessible in production. From there, they injected a 23-line JavaScript skimmer into the payment processing page that captured every field users typed: name, card number, expiry, CVV. 380,000 payment cards were stolen over 2 weeks before the breach was detected. The script was designed to exfiltrate data to a third-party domain that appeared legitimate (baways.com — a typosquatted domain). BA was fined £20M under GDPR. The attack vector was not a zero-day — it was a path an attacker found by guessing URLs with a wordlist. Directory busting of exactly the kind you'll practice today was the entry point for one of the largest web-based data thefts in European history.
// attacker vs defender mindset
⚔ Attacker
  • Run gobuster with medium wordlist first — fast pass to find admin panels, API endpoints, and hidden dev paths.
  • Then run ffuf with extension filters — look for .php, .asp, .env, .bak, .sql files specifically (Magecart's exact method).
  • Use Wappalyzer to detect the CMS version — if it's Wordpress 4.x (old), there are known plugin vulns to chain.
  • Check /.well-known/ paths — might reveal security.txt, apple-app-site-association, or assetlinks.json.
🛡 Defender
  • Run these same tools against your own website to find what attackers would find — if you find /admin, so will they.
  • Remove or require authentication for any staging, dev, or admin path that doesn't need to be public.
  • Block automated scanning via WAF rate-limiting and return 404 for non-existent paths instead of 403 (which confirms existence).
  • Monitor 404 spikes in access logs — a burst of 4XX from one IP is a directory bruteforce in progress.
HANDS-ON LAB — DIRECTORY BUST METASPLOITABLE + A REAL DOMAIN

Two targets today — Metasploitable's web server (safe, local) and a real website you have permission to scan (your own or a bug bounty program).

  1. Install SecLists on Kali: sudo apt update && sudo apt install seclists -y. This gives you 30+ specialized wordlists.
  2. Run gobuster against Metasploitable's web server (port 80): gobuster dir -u http://192.168.56.102 -w /usr/share/wordlists/dirb/common.txt. Note every path found.
  3. Run the same scan with a broader wordlist (SecLists medium): gobuster dir -u http://192.168.56.102 -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt. Compare results — the difference should surprise you.
  4. Run ffuf against Metasploitable looking for specific extensions: ffuf -u http://192.168.56.102/FUZZ -w /usr/share/wordlists/dirb/common.txt -e .php,.txt,.bak,.zip,.sql
  5. Install Wappalyzer (browser extension) or use WhatWeb: whatweb 192.168.56.102. Compare the tech stack it identifies against what you know about Metasploitable.
  6. Pick a real site you own (or use tryhackme.com) and run WhatWeb against it — see what information you're leaking about your tech stack to anyone who scans.

INTERVIEW READY
Q: "How would you map a web application's attack surface before testing it?"
A: "I start with passive reconnaissance — WhatWeb and Wappalyzer to identify the technology stack: server type, CMS, framework versions, and any WAF present. If the site is behind Cloudflare, that changes my approach. Then I move to active recon with gobuster using a medium wordlist from SecLists to discover hidden directories and files — admin panels, API endpoints, backup archives. I follow up with ffuf extension filtering targeting .php, .bak, .sql specifically. The British Airways Magecart breach in 2018 is a direct example of why this matters — attackers found an unsecured staging path via directory enumeration and injected a credit card skimmer that stole 380,000 payment records. After mapping the surface, I'd have a prioritized list of entry points for the exploitation phase."
AI TOOLS
ChatGPT + WhatWeb output
Paste your WhatWeb fingerprint and ask "what are the known vulnerabilities for Apache 2.2.8 / PHP 5.2.4?" — instant prioritized CVE list for your target.
FFuf + AI-generated wordlists
Ask an LLM to generate a custom wordlist based on your target's context — e.g. "generate 50 possible admin panel paths for a Laravel application."
Wappalyzer CLI + automation
Feed multiple targets to Wappalyzer's API and auto-categorize sites by tech stack. Use with Claude to build a target attack playbook per technology profile.
Gobuster + Notebook
Run multiple gobuster scans with different wordlists using a bash loop, pipe results into an AI summary: "of the 47 paths found, prioritize the 7 with login forms or upload functionality."

ENTREPRENEUR ANGLE

Web recon is the most sellable entry-level security service: "We'll scan your website and tell you exactly what hidden paths an attacker can find." Small businesses often have old admin panels, forgotten test directories, and exposed config files that a gobuster scan can find in minutes. A basic website attack surface audit — technology fingerprint + directory enumeration + findings report — is a ₹5,000–₹15,000 deliverable you can run in an afternoon with exactly the tools you learned today. The BA fine of £20M under GDPR for a breach that started with directory enumeration is the sales pitch: "For the cost of one vulnerability scan, you avoid the risk of a GDPR fine 1000x that amount."

TEACH IT BACK

Explain to a non-technical person: "Imagine a building. You can see the main entrance (the homepage). But there are side doors, roof hatches, basement stairs, and unmarked doors hidden in the corridors. Directory busting is systematically trying every doorknob to see which ones open. Technology fingerprinting is looking at the building materials — steel frame vs wood, brand of lock, alarm system model — to figure out the best way to get in if a door is unlocked. British Airways' attacker found a forgotten side door in their website that should have been locked, walked in, and installed a card skimmer that stole 380,000 credit cards. We're finding the side doors before the bad guys do."
