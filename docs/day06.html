Networking I — the OSI model

Every packet, every exploit, every Wireshark capture you'll ever look at maps onto 7 layers. Today you stop saying "the network" as one blob and start seeing the stack.

THEORY

The OSI (Open Systems Interconnection) model is the universal framework for how data moves across a network — 7 layers, each adding its own piece of information as data travels down, then unwrapping it on the way back up (a process called encapsulation).

L7
Application
HTTP, DNS, SMB — what users/programs directly interact with
L6
Presentation
Encryption (TLS), data formatting/translation
L5
Session
Opens/maintains/closes the connection session
L4
Transport
TCP (reliable) vs UDP (fast) — ports live here
L3
Network
IP addresses, routing between networks
L2
Data Link
MAC addresses, switches, ethernet frames
L1
Physical
Cables, signals, raw bits — the actual hardware

Mnemonic: "Away Pizza Saving Throws Near Drunk People" (L7→L1) — or build your own, it sticks better.

// real, named incident — across multiple layers
2017 — WANNACRY / ETERNALBLUE
A multi-layer attack, textbook example
WannaCry spread using EternalBlue, an exploit in Microsoft's SMB protocol (Layer 7 — Application). It traveled over TCP port 445 (Layer 4 — Transport), routed across networks via IP (Layer 3 — Network), infecting 200,000+ computers in 150+ countries within days, including the UK's NHS. Understanding which layer a vulnerability lives at tells you exactly what kind of defense actually stops it — in this case, blocking port 445 at the firewall (Layer 4) and patching SMB (Layer 7) both mattered.
// attacker mindset vs. defender mindset
⚔ How an attacker thinks
  • "Which layer has the weakest control here — can I spoof a MAC address (L2), forge an IP (L3), or exploit the application protocol itself (L7)?"
  • Lower-layer attacks (ARP spoofing, MAC flooding) are often invisible to application-layer security tools entirely.
  • EternalBlue worked because the protocol itself (SMBv1) was flawed — no amount of "be careful what you click" would've stopped it.
🛡 How a defender thinks
  • "Defense needs a control at every layer — firewall rules (L3/4), encryption (L6), and patched application protocols (L7) — one layer alone is never enough."
  • Knowing which layer an alert came from tells you which team/tool should respond.
  • Disabling legacy protocols (like SMBv1) proactively removes entire categories of future exploits.

INTERVIEW READY
Q: "Why does the OSI model matter in practice, not just theory?"
A: "It tells you where to look and what tool to reach for. WannaCry in 2017 is a clean example — the actual flaw (EternalBlue) lived in SMB at Layer 7, but it traveled over TCP port 445 at Layer 4 and routed via IP at Layer 3. That meant defense needed action at multiple layers: blocking port 445 at the firewall, and patching the SMB protocol itself. If I only think 'the network is broken' without knowing which layer, I can't pick the right fix."
ENTREPRENEUR ANGLE

When you eventually explain a security gap to a non-technical client, the OSI model gives you a simple way to translate: "your firewall protects layer 3 and 4, but layer 7 — your actual web application — has separate vulnerabilities that need separate testing." This kind of layered explanation makes a freelance security assessment sound thorough and credible rather than vague, and naturally upsells a client from "just a firewall check" into a fuller audit package — exactly the kind of conversation that grows a one-off gig into a retained client for your future firm.

TEACH IT BACK

Explain to a non-technical person: "Sending data across a network is like mailing a letter through several layers of packaging — a letter (Application), sealed in an envelope (Presentation), placed in a labeled box (Session), with a shipping label and tracking number (Transport), routed through sorting centers by address (Network), carried by a specific truck (Data Link), driving on an actual road (Physical). WannaCry in 2017 exploited a flaw in how one specific 'package label' (the SMB protocol) was handled — and because nearly everyone used the same mail system, it spread to 150+ countries in days."
