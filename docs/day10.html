Python for Security I — syntax & first scripts

Python is the #1 language in cybersecurity — used by attackers, defenders, and bug bounty hunters alike. Today you learn the syntax that matters and write your first actual security-oriented scripts.

THEORY

Why Python for cybersecurity? It's fast to write, easy to read, has massive security libraries (Scapy, Requests, Paramiko, Pwntools), and runs natively on Linux. Every major security tool category — port scanners, exploit frameworks, malware, OSINT tools — has a Python implementation. You don't need to be a developer; you need to be able to read and write short, purposeful scripts.

The security use cases you'll hit constantly: reading/parsing files (log analysis), making HTTP requests (web app testing), socket programming (port scanners, reverse shells), automating repetitive recon tasks, and writing quick CTF solve scripts.

Core syntax you must know cold:

Variables & Types
ip = "192.168.1.1"
port = 80
alive = True
No type declaration needed. Strings, ints, booleans — just assign.
if / else
if port == 80:
    print("HTTP open")
elif port == 443:
    print("HTTPS open")
else:
    print("Other")
Indentation IS the syntax — no braces. Get it wrong and code breaks.
Loops
ports = [21,22,80,443]
for p in ports:
    print(f"Port: {p}")
Loop over a list of ports, IPs, or URLs — the heart of any scanner script.
Functions
def scan(ip, port):
    # logic here
    return result

scan("10.0.0.1", 80)
Define once, reuse anywhere. Keeps your scripts clean and testable.
File I/O
with open("ips.txt","r") as f:
    for line in f:
        print(line.strip())
Reading a wordlist, IP list, or log file — used in almost every security script.
Key Libraries
import socket
import requests
from scapy.all import *
socket = raw network, requests = HTTP, scapy = packet crafting.
One rule: Python indentation is not optional or stylistic — it defines code blocks. A wrong indent = a broken script. Always use 4 spaces (not tabs) in security tools.
// real named incident
2022–2023 — PYPI SUPPLY CHAIN ATTACKS
Malicious Python packages downloaded by thousands of developers
In 2022 and 2023, attackers repeatedly uploaded malicious packages to PyPI (Python's official package index) with names intentionally similar to legitimate libraries — a technique called "typosquatting." For example, packages like request (vs the real requests) or colourama (vs colorama) were downloaded by developers who made a one-letter typo in their pip install command. The malicious packages executed Python code on install that stole crypto wallets, Discord tokens, and browser credentials. This is directly why you always verify package names before pip installing anything — and why reading Python code before running it is a real security skill, not just a developer habit.
// attacker vs defender mindset
⚔ Attacker
  • Python scripts automate what would take hours by hand — scanning 1,000 hosts, fuzzing 10,000 URLs, brute-forcing a login — in minutes.
  • Most real-world malware drops Python scripts because they run on any OS with Python installed — no compilation needed.
  • PyPI typosquatting shows even the install command can be the attack vector.
🛡 Defender
  • Reading Python code is a core malware analysis skill — most modern malware scripts are readable once you know the syntax.
  • Python automates log parsing, alert triage, and threat hunting — the same scripts that help attackers help defenders scale too.
  • Always pin package versions in requirements.txt and verify checksums — PyPI attacks exploit developers who don't.
