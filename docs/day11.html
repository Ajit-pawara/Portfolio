Python for Security II — sockets & automation

Day 10 gave you the syntax. Today you use it for real — building network tools from scratch using Python sockets, the same library that powers everything from port scanners to reverse shells to the Mirai botnet.

THEORY

What is a socket? A socket is a programming interface that lets your Python script open a real network connection — the same way a browser connects to a website, or Nmap connects to a port. Think of it as a phone handset in code: you dial (connect), talk (send data), listen (receive data), and hang up (close).

Two types you need to know:
socket.SOCK_STREAM = TCP socket — reliable, ordered, connection-based. Use for web, SSH, anything that needs guaranteed delivery.
socket.SOCK_DGRAM = UDP socket — fast, connectionless, fire-and-forget. Use for DNS queries, speed-critical recon.

The four socket actions every script uses:

s = socket.socket()
Creates the socket object. AF_INET = IPv4. SOCK_STREAM = TCP.
s.connect((ip, port))
Opens the TCP connection — completes the 3-way handshake you learned on Day 7.
s.send(data.encode())
Sends data as bytes. You must encode strings — sockets speak bytes, not strings.
s.recv(1024)
Receives up to 1024 bytes of response. Decode with .decode() to read as text.

Automation pattern — why this scales: once you can open one socket connection in Python, a for loop makes it scan 65,535 ports. Two nested loops make it scan an entire subnet. This is literally how network scanners, brute-forcers, and botnets work — the same 10 lines of socket code, looped intelligently.

Important: always use s.settimeout() in your scripts — without it, a connection attempt to a closed/filtered port can hang forever and your script freezes. 0.5–1 second is standard for scanning scripts.
// real named incident
2016 — MIRAI BOTNET SOURCE CODE RELEASE
Python socket automation at internet scale — used to take down half the internet
Mirai was a botnet that automated exactly what you're learning today — socket connections at massive scale. It scanned the entire IPv4 internet (every public IP address) for open Telnet ports (port 23) using concurrent socket connections, then attempted login with 61 default username/password combinations. Devices that accepted the defaults (routers, cameras, DVRs) were absorbed into the botnet. The result: the October 2016 Dyn DNS attack (from Day 1!) that took Twitter, Netflix, and Reddit offline for hours. The Mirai source code was released publicly in 2016 and immediately spawned hundreds of variants. Every one of them built on the same socket automation concept you're writing today — the difference is authorization and intent.
// attacker vs defender mindset
⚔ Attacker
  • Automation is the real weapon — scanning 65,535 ports by hand is impossible, but a Python socket loop does it in minutes.
  • Mirai's genius was concurrent scanning — not waiting for one connection before trying the next, but opening thousands simultaneously.
  • Default credentials are still an attack surface in 2026 — most IoT devices still ship with admin/admin or similar. Scripted credential checks are trivial.
🛡 Defender
  • Your own Python scripts can scan your infrastructure for open ports and default credentials before an attacker does — same tool, authorized context.
  • Detecting Mirai-style scans: watch for one source IP attempting connections to hundreds of different IPs on the same port in a short window — that's a scanner signature.
  • Rate limiting and connection throttling are the network-level defenses against automated socket-based scanning.
