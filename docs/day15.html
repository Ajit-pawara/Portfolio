// Mini Quiz — 15 Questions, Say Answers Out Loud

Cover the answer. Say it out loud. If you can't answer fluently, mark it and revisit that day before Day 16.

Q01
A ransomware attack encrypts a hospital's files and demands payment. The hospital can still read old unencrypted backups. Which CIA element(s) were broken?
Answer:
Availability (files inaccessible) + potentially Confidentiality if data was exfiltrated before encryption. Integrity is intact — data wasn't altered, just encrypted/locked.
Q02
You find a binary at /tmp/kworker running as root. Why is this suspicious, and what command confirms whether it's real?
Answer:
Real kernel threads are in brackets [kworker] in ps and live in kernel space — not in /tmp. Confirm with: ls -l /proc/[PID]/exe — a real kernel thread has no file path, malware will show the actual binary.
Q03
Convert rwxr-x--- to octal notation.
Answer:
750. Owner=7(rwx), Group=5(r-x), Others=0(---).
Q04
WannaCry exploited EternalBlue. Name the OSI layer of the vulnerability, the transport layer port it spread through, and two required defenses.
Answer:
Vulnerability at L7 (SMBv1 application protocol). Spread via TCP port 445 (L4). Two defenses: (1) patch/disable SMBv1 at L7, (2) block TCP port 445 at firewall (L4). One alone was insufficient.
Q05
What is the broadcast address and how many usable hosts exist for the subnet 10.0.5.0/24?
Answer:
Broadcast: 10.0.5.255. Usable hosts: 254 (10.0.5.1 through 10.0.5.254). Network address (10.0.5.0) and broadcast (10.0.5.255) are not assignable.
Q06
A developer runs `pip install requets` and their system is immediately compromised. Explain the attack mechanism.
Answer:
PyPI typosquatting (2022–23 attacks). "requets" is a malicious package with a name one letter different from the real "requests". The malicious setup.py executes on pip install — before any code is imported or run. The attack is in the install command itself.
Q07
Name 4 Windows Event IDs a SOC analyst monitors and what each detects.
Answer:
4624 = successful logon. 4625 = failed logon (brute force). 4688 = process creation (disabled by default — catch living-off-the-land). 1102 = audit log cleared (attacker covering tracks — this fires even when they try to hide it).
Q08
Lazarus Group stole $81M from Bangladesh Bank using a standard Kali Linux tool. Which tool, and why was it effective?
Answer:
Wireshark — a packet capture tool. They installed it on a compromised host inside the bank and passively captured SWIFT transaction traffic for weeks. It was effective because internal bank traffic was unencrypted, and Wireshark generates no network traffic itself — it just listens silently.
Q09
What Nmap flag runs vulnerability detection scripts, and which specific output told you vsftpd 2.3.4 was exploitable?
Answer:
Flag: nmap --script vuln [ip]. Output: "ftp-vsftpd-backdoor: VULNERABLE" — Nmap's NSE script specifically checks for the vsftpd 2.3.4 backdoor (CVE-2011-2523) and confirms it before Metasploit even opens.
Q10
Write a one-line Python socket code snippet that attempts a TCP connection to port 80 on 192.168.1.1 with a 1 second timeout.
Answer:
s = socket.socket(); s.settimeout(1); s.connect(("192.168.1.1", 80)) — wrapped in try/except to handle closed/filtered ports cleanly.
Q11
What is the Cyber Kill Chain stage where attackers spent most of their effort in the 2013 Target breach, and what did they do there?
Answer:
Reconnaissance — they identified and targeted a third-party HVAC vendor with access to Target's network, then phished that vendor's credentials. The actual lateral movement to POS systems came much later, after weeks of quiet internal observation.
Q12
Why is the VirtualBox Host-Only network setting critical for your home lab, vs NAT or Bridged?
Answer:
Host-Only creates an isolated network between VMs only — no internet access from either VM. NAT shares the host's internet (VMs could reach real targets accidentally or malware could call home). Bridged exposes VMs to your real local network (other devices, routers, etc.). Host-Only = safe, contained, legal to attack freely.
Q13
SolarWinds SUNBURST went undetected for 9 months. Name two Windows logging configurations that, if active, would have significantly increased the chance of earlier detection.
Answer:
Event ID 4688 (process creation logging — disabled by default) and Event ID 4104 (PowerShell script block logging — also disabled by default). These are enabled via Group Policy and cost nothing — their absence let SUNBURST's process activity go unlogged across thousands of organizations.
Q14
What is the difference between TCP (SOCK_STREAM) and UDP (SOCK_DGRAM) sockets, and which does Nmap use for a SYN scan?
Answer:
TCP (SOCK_STREAM) = reliable, ordered, connection-based — requires the 3-way handshake. UDP (SOCK_DGRAM) = connectionless, fire-and-forget, no handshake. An Nmap SYN scan (-sS) uses raw TCP — sends SYN but never completes the handshake (no ACK), which is why it's "stealth" — most firewalls and application logs only record fully established connections.
Q15
Name three specific files or directories on a Linux system that an attacker checks immediately after getting a shell, and why each one matters.
Answer:
/etc/shadow — contains password hashes for all users (needs root to read, so getting it means you have privilege or found a path to escalate). /home/[user]/.bash_history — shows commands the user ran previously, may contain credentials, IPs, or clues about the system. /tmp — world-writable, check if other attackers already dropped payloads here, and use it to stage your own tools temporarily without needing write access elsewhere.
