Kali Linux Tools — Nmap, Wireshark, Metasploit, Burp Suite

Day 13 built the lab. Today you actually use it — four tools that appear in every professional penetration test, every SOC investigation, and almost every technical interview for a security role.

THEORY

Every security engagement uses a small core set of tools repeatedly. Today's four are the industry's most recognized — used by attackers, defenders, and everything in between. The goal today is not to master each one — it's to run each one against Metasploitable and understand what you're seeing.

Nmap
Network Scanner / Recon
Maps open ports, services, OS, and version information across any IP range. The first tool run in almost every penetration test — "what's here and what's running?"
Wireshark
Packet Analyzer
Captures and reads every packet on a network interface in real time. Used by attackers to intercept traffic AND by defenders to analyze incidents. Lazarus Group used it to steal $81M.
Metasploit
Exploitation Framework
Database of exploits for known vulnerabilities. You pick a target service, choose an exploit, run it. Pre-built modules mean you can test a known CVE in 5 commands. Used extensively in authorized pentests.
Burp Suite
Web App Proxy / Tester
Sits between your browser and a website, intercepting every HTTP request. Lets you modify, replay, and analyze web traffic. The standard tool for web application security testing.

Nmap switches you must know for any interview or lab:

nmap -sV [ip]
Service version detection — shows exactly what software is running on each port.
nmap -sS [ip]
SYN scan (stealth) — sends SYN, receives SYN-ACK, but never completes handshake. Faster, less likely to appear in logs.
nmap -O [ip]
OS detection — guesses the target operating system from network behavior.
nmap -A [ip]
Aggressive scan — combines -sV, -O, traceroute, and NSE scripts. Loud but comprehensive.
nmap --script vuln [ip]
Run Nmap's vulnerability detection scripts against the target — quick CVE check in one command.
nmap -p- [ip]
Scan all 65,535 ports instead of the default top 1,000. Slower but finds hidden services.
Wireshark key filters for security work: http (show only HTTP traffic), tcp.port == 80 (specific port), ip.addr == 192.168.56.102 (single host), tcp.flags.syn == 1 (see SYN packets — spot scanners). These four cover 80% of basic traffic analysis.
// real named incident
2016 — BANGLADESH BANK SWIFT HEIST — $81M STOLEN USING WIRESHARK
Lazarus Group used Wireshark to study bank traffic for weeks before stealing $81 million
In 2016, the Lazarus Group (North Korean state-sponsored attackers) compromised computers inside Bangladesh's central bank and installed Wireshark — a standard, legitimate network analyzer — to capture and analyze SWIFT financial transaction traffic in real time. They observed the exact format, authentication flow, and timing of legitimate SWIFT messages for weeks without triggering any alerts. Once they understood the protocol completely, they crafted fraudulent SWIFT transfer requests that successfully moved $81 million to accounts in the Philippines before the fraud was caught (another $850M was blocked due to a spelling error in one of the transfer requests). Wireshark didn't "hack" anything — it just listened. That listening turned into the largest bank cyber-theft in history at the time. This is why unencrypted internal traffic is dangerous even on your "private" network — any compromised host with a packet sniffer becomes a full surveillance point.
// attacker vs defender mindset
⚔ Attacker
  • Nmap first — always. "What's running?" before anything else. vsftpd 2.3.4 on Metasploitable from Day 13 has a known backdoor — Nmap + Metasploit finds and exploits it in under 5 minutes.
  • Wireshark on a compromised host gives you everything the victim sends in plaintext — credentials, session tokens, internal protocol details.
  • Burp Suite intercepts web logins — change the username or parameter value before it reaches the server. No code needed.
🛡 Defender
  • Nmap your own perimeter regularly — find what's exposed before an attacker does. "Know thy network" is the first rule of defense.
  • Wireshark in a SOC context: when something suspicious happens, capture traffic on the affected segment and look for anomalies — C2 beaconing, data exfiltration, credential harvesting.
  • Encrypt internal traffic. Lazarus's entire Wireshark campaign was possible because SWIFT traffic inside the bank was unencrypted — they read it like a book.
HANDS-ON LAB — ALL FOUR TOOLS IN YOUR HOME LAB

Run these in order against Metasploitable (your legal target from Day 13). Each builds on the last.

  1. Go to tryhackme.com/room/nmap — complete the Nmap scan types section to get TryHackMe lab credit alongside your home lab practice.
  2. Run the Nmap commands below against Metasploitable — execute the aggressive scan and the vulnerability script scan specifically.
  3. Start Wireshark on Kali while running an Nmap scan — watch the SYN packets fly in real time. Apply the filter tcp.flags.syn == 1 and see your own scan pattern.
  4. Launch Metasploit and exploit vsftpd 2.3.4 on Metasploitable using the script below — this is your first real exploit against a real service.
  5. Open Burp Suite, configure your browser to use it as a proxy (127.0.0.1:8080), browse to the Metasploitable DVWA web app at its IP on port 80, and intercept one login request.

INTERVIEW READY
Q: "Walk me through how you'd approach a penetration test at a high level."
A: "Starting with Nmap — I'd run a service version scan to identify what's running and on which ports, then use the vulnerability script to flag known CVEs immediately. Anything interesting gets investigated with Burp Suite if it's web-facing, or Metasploit if it's a network service with a known exploit. Throughout I'd have Wireshark running on my interface to capture traffic — both to understand the network's communication patterns and to capture any cleartext credentials in transit. The Bangladesh Bank SWIFT heist is the clearest example of why traffic capture matters even inside a 'trusted' network — Lazarus Group used Wireshark in passive mode for weeks and learned an entire banking protocol just by listening."
AI TOOLS
ChatGPT + Nmap output
Paste your full Nmap scan output and ask "what are the highest-priority vulnerabilities here and which Metasploit modules target them?" Instant threat analysis.
Wireshark + AI filters
Describe suspicious traffic in plain language to ChatGPT and ask it to write the Wireshark display filter — e.g., "find all traffic going to port 443 except from my IP."
Burp Suite AI (Pro)
Burp Suite Professional includes an AI-assisted scanner. Free community edition covers manual testing — the Pro AI extension auto-finds OWASP Top 10 vulnerabilities.
Metasploit + AI
Ask Claude/ChatGPT "which Metasploit module targets vsftpd 2.3.4 and what options does it need?" — faster than searching exploit-db manually during a lab session.

ENTREPRENEUR ANGLE

These four tools together form the basic toolkit of a freelance penetration test. An authorized scan of a small business's network perimeter (Nmap for exposure, Burp for their website, Wireshark for internal traffic hygiene, Metasploit for known CVE validation) is a real, deliverable service at ₹10,000–₹30,000 for a small engagement. Every tool you learn today is a line item in your future firm's service catalogue. Knowing Bangladesh Bank lost $81M to a passive Wireshark install makes "we'll check if your internal traffic is encrypted" an easy sell.

TEACH IT BACK

Explain to a non-technical person: "Nmap is a door-checker — it walks up to every door in a building and notes which ones are open and what's on the other side. Wireshark is a wire-tap — it listens to every conversation in the building without joining them. Metasploit is a skeleton-key collection — once Nmap finds a known weak lock, Metasploit has the key for it. Burp Suite is a letter-opener — it reads your mail before it gets delivered, lets you modify it, then reseals it. Lazarus Group used the wire-tap (Wireshark) to learn the exact language of Bangladesh Bank's internal conversations, then sent fake letters (Burp-style HTTP manipulation) that looked identical to real ones — and walked out with $81 million."
