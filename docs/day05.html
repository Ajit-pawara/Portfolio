Bash Scripting Basics — automating the boring stuff

A one-liner you type once is a habit. A script you write once runs a thousand times. Today you write your first real scripts — and learn why a single Bash bug once broke half the internet.

THEORY

A bash script is just a text file of commands you'd otherwise type one at a time, run together as a program. Every script starts with a shebang: #!/bin/bash, telling the OS which interpreter to use.

$1 $2 $#
First arg, second arg, total argument count passed to the script.
read name
Pauses the script and stores typed input into the variable "name".
if [ "$x" -eq 10 ]
Conditional logic — note the required spaces inside the brackets.
chmod +x script.sh
Makes the script executable so you can run it with ./script.sh
// real, named incident — not hypothetical
2014 — SHELLSHOCK (CVE-2014-6271)
The Bash bug that broke the internet
A flaw in how Bash parsed environment variables let attackers attach malicious code to a crafted variable and have it execute automatically — no password, no login needed, on any system using vulnerable Bash through CGI scripts, DHCP clients, or SSH. Within 24 hours of disclosure, botnets were actively scanning the internet for vulnerable servers. It affected an estimated 500 million+ devices and is still referenced as one of the most severe vulnerabilities ever found, precisely because Bash itself is everywhere.
// attacker mindset vs. defender mindset
⚔ How an attacker thinks
  • "Can I automate scanning thousands of targets for one known flaw at once?" — Shellshock-hunting scripts spread within hours of disclosure.
  • Bash scripts are how recon, exploitation, and post-exploitation get repeated at scale instead of done by hand.
  • A script that works on one box should work on a hundred — automation is the actual force multiplier in offensive security.
🛡 How a defender thinks
  • "Can I script my own patch-checking and log review instead of doing it manually every day?"
  • Shellshock's lesson: patch fast, and know exactly which systems run which Bash version — a script can check this across a whole fleet in seconds.
  • Defensive automation (backups, log rotation, alerting) is just as valuable a skill as offensive scripting.

ENTREPRENEUR ANGLE

Small businesses running their own Linux servers almost never have basic automated health checks — patch status, disk space, failed login attempts. A simple bash script bundle ("automated daily security check-in email") is a low-effort, high-value freelance product: you write it once, deploy it to multiple clients, and charge a small recurring fee for "monitoring." This is exactly the kind of repeatable, scriptable service that scales into your future firm without you manually doing the same check by hand for every client.

TEACH IT BACK
