What is Cybersecurity + The CIA Triad
Equifax 2017 · Stuxnet 2010 · Dyn/Mirai 2016
// core concept
Confidentiality
Only authorized people can see the data. Breaking it = data leak. Equifax 2017 — 147M records exposed.
Integrity
Data isn't altered without authorization. Breaking it = silent tampering. Stuxnet 2010 — centrifuge readings falsified.
Availability
Systems are usable when needed. Breaking it = downtime/denial. Dyn DNS/Mirai 2016 — Twitter, Netflix offline for hours.
// key rule

Every attack you will ever study breaks one (or more) of these three. When you read a breach headline, your first question should always be: which of the CIA triad did this break?

// interview answer
Q: Walk me through the CIA Triad with a real example.
Equifax 2017 = Confidentiality (147M records exposed, nothing changed). Stuxnet 2010 = Integrity (false sensor data fed to operators). Dyn/Mirai 2016 = Availability (DDoS took major sites offline, nothing stolen). Every security decision traces back to protecting one of these three.

03
Linux Fundamentals I — Filesystem & Permissions
PwnKit CVE-2021-4034
// critical directories
/etc/passwd & /etc/shadow
User accounts + password hashes. shadow is root-only — getting here = target for attackers.
/var/log/
System logs — where defenders spend most of their investigation time.
/tmp/
World-writable. Attackers drop payloads here because every user can write to it.
/root/
Superuser's home. Reaching here = full system compromise.
// permission numbers — memorize these
r = 4, w = 2, x = 1
rwxr-xr-- = 754. Owner=7(rwx), Group=5(r-x), Others=4(r--)
chmod 600 file
Only owner can read/write. Correct for sensitive files like SSH keys.
chmod 777 file
Everyone can read/write/execute. NEVER do this in production — shows up in breach reports constantly.
SUID bit (4000)
File runs as its owner (often root). PwnKit CVE-2021-4034 exploited a SUID binary for 12+ years.
// key command
find / -perm -4000 -type f 2>/dev/null
Find all SUID binaries on the system — first thing attackers run post-compromise for privilege escalation.
04
Linux Fundamentals II — Users, Processes & Shell
TeamTNT cryptojacking 2020–21
// key concepts
su vs sudo
su switches user entirely. sudo runs one command as root AND logs it. Sudo is safer — logs are evidence.
PID / PPID
Every process has an ID (PID) and parent ID (PPID). A "kernel thread" with a normal user as its PPID is suspicious.
ps aux
Snapshot of all running processes. TeamTNT renamed malware to "kworker" to hide in plain sight here.
kill [PID]
Sends SIGTERM. kill -9 [PID] sends SIGKILL — can't be ignored, forceful termination.
// the real lesson

Process names are lies. TeamTNT's cryptomining malware renamed itself to look like a legitimate kernel thread (kworker). Always verify with ls -l /proc/[PID]/exe — the actual binary path doesn't lie, the name field does.

// commands
ps aux | grep -v "\[" | grep kworker
Find fake kernel threads — real ones are in brackets, malware isn't.
ls -l /proc/[PID]/exe
Reveal the actual binary a process is running — bypasses name spoofing.
05
Bash Scripting Basics
Shellshock CVE-2014-6271
// must-know syntax
#!/bin/bash
Shebang — must be line 1 of every script. Tells OS which interpreter to use.
$1 $2 $#
First arg, second arg, total argument count passed to the script.
if [ "$x" -eq 10 ]
Spaces inside brackets are required. Get it wrong and script errors silently.
chmod +x script.sh
Makes script executable. Without this, ./script.sh gives "Permission denied."
// real incident

Shellshock (CVE-2014-6271, 2014): a flaw in how Bash parsed environment variables let attackers append malicious code that executed automatically — no login needed. 500M+ devices affected. Botnets scanning for it within 24 hours of disclosure. Lesson: Bash is everywhere, and flaws in it are catastrophic in scope.

// your first real security script
#!/bin/bash
echo "User: $(whoami)"
echo "Host: $(hostname)"
ss -tuln
sysinfo.sh — your first recon script. Shows user, hostname, and all listening ports. Extends to 100+ hosts with a loop.
06
Networking I — The OSI Model
WannaCry/EternalBlue 2017
// 7 layers — memorize top to bottom
L7 — Application
HTTP, DNS, SMB — what users touch. WannaCry exploited SMBv1 here.
L6 — Presentation
Encryption (TLS), formatting, compression.
L5 — Session
Opens, maintains, closes connection sessions.
L4 — Transport
TCP vs UDP. Ports live here. WannaCry spread via TCP port 445.
L3 — Network
IP addresses, routing between networks. WannaCry routed via IP to 150+ countries.
L2 — Data Link
MAC addresses, switches, ethernet frames.
L1 — Physical
Cables, signals, raw bits — actual hardware.
// the real lesson

Knowing which OSI layer a vulnerability lives on tells you what defense actually stops it. WannaCry required action at L7 (patch SMBv1) AND L4 (firewall block TCP port 445). Fixing only one layer is not enough.

07
Networking II — TCP/IP & Ports
MafiaBoy SYN Flood 2000
// TCP three-way handshake
1. SYN →
"I want to connect." Your machine sends a connection request.
2. SYN-ACK ←
"OK, I'm ready." Server acknowledges and prepares resources.
3. ACK →
"Confirmed, let's go." Connection established, data flows.
// ports to memorize
21 — FTP
File transfer. Often unencrypted — common pentest target.
22 — SSH
Secure remote shell. Encrypted. Brute-forced constantly if exposed.
80 — HTTP
Plain web traffic — readable by anyone on same network.
443 — HTTPS
Encrypted web traffic. The padlock icon in your browser.
3389 — RDP
Windows Remote Desktop. Very commonly attacked if internet-exposed.
// SYN flood + real incident

MafiaBoy (Feb 2000): a 15-year-old flooded Yahoo, eBay, CNN, Amazon with SYN requests (step 1) and never sent step 3 (ACK). Servers held resources open for incomplete connections until exhausted and crashed. ~$1.2B in damages. Still a valid attack today without SYN cookie protection.

08
Networking III — DNS, HTTP/HTTPS & DHCP
Baidu DNS Hijack 2010
// DNS lookup chain (6 steps)
1. Local Cache
Already looked this up? Use cached result — skip everything.
2. Recursive Resolver
Your ISP's lookup helper — asks on your behalf.
3. Root Server
Doesn't know the IP — points to TLD server.
4. TLD Server
Manages .com/.in/.org — points to authoritative server.
5. Authoritative Server
The domain's own server — returns the actual IP.
6. Cache + Connect
IP cached locally for next time, browser connects.
// HTTP vs HTTPS
HTTP (port 80)
Plain text. Anyone on the same network can read everything — passwords, cookies, all of it.
HTTPS (port 443)
TLS-encrypted. Intercepted traffic is unreadable garbage. Certificate verifies the server is real.
// real incident

Baidu DNS Hijack (2010): attackers poisoned the DNS registrar records for China's largest search engine. 100M+ users redirected for hours to a page controlled by "Iranian Cyber Army" — without touching Baidu's servers at all. Just changed the phone book entry.

// commands
nslookup google.com
See which DNS server resolves it and what IP it returns.
dig tryhackme.com
Full lookup chain with TTL — how long the result is cached.
cat /etc/resolv.conf
Which DNS server your machine uses. 1.1.1.1 or 8.8.8.8 is safer than ISP default.

Subnetting & Routing Basics
Target Corporation 2013
// CIDR — memorize these
/24 = 255.255.255.0
254 usable hosts. Most common in home/small office networks.
/16 = 255.255.0.0
65,534 usable hosts. Large enterprise internal networks.
/30 = 255.255.255.252
2 hosts only. Point-to-point router links.
/32
Single host. Used in firewall rules to specify one exact IP.
// subnet anatomy for 192.168.1.0/24
Network: 192.168.1.0
Identifies the subnet — never assigned to a device.
Broadcast: 192.168.1.255
Reaches all devices on subnet — never assigned either.
Usable: .1 → .254
254 devices can be assigned here.
Gateway: 192.168.1.1
The router — traffic outside the subnet goes here first.
// real incident

Target 2013: attackers entered via an HVAC vendor's credentials, moved laterally directly to POS systems because the network was flat (no segmentation). Proper subnetting with firewall rules between vendor zone and payment systems would have contained the breach. 40M card numbers stolen, $18.5M settlement.

// commands
ip addr show
Your IP and subnet mask — what network you're on.
ip route show
Your routing table — where traffic goes for each destination.
traceroute google.com
Every router hop from you to the destination — visualizes routing in action.
nmap -sn 192.168.1.0/24
Discover all live hosts on your subnet. First thing attackers do post-compromise.

Python for Security I — Syntax & First Scripts
PyPI Supply Chain 2022–23
// syntax you must know cold
Variables
ip = "192.168.1.1" — no type declaration. String, int, bool — just assign.
If/else
Indentation IS the syntax — no braces. Wrong indent = broken code silently.
For loop
for p in [80,443]: print(p) — heart of any scanner or automation script.
Functions
def scan(ip,port): — define once, reuse anywhere, keeps scripts clean.
File I/O
with open("ips.txt","r") as f: — reads wordlists, IP lists, log files.
Socket library
import socket — raw network connections. Built your first port scanner with this.
// real incident

PyPI Supply Chain (2022–23): attackers uploaded packages named one letter different from real ones (request vs requests). Developers with a typo in pip install downloaded and ran credential-stealing Python automatically. Reading Python code is now a defensive skill — not just a developer one.

// your first security script (port_checker.py)
import socket
Raw network library — socket.connect() attempts TCP handshake to check if port is open.
s.settimeout(1)
Don't wait forever for a closed port — 1 second timeout keeps the script fast.
scanme.nmap.org
Legal test target — Nmap's official scan-me host. Safe to test your scanner against.

// Quick Self-Quiz — Can You Answer These Without Looking?

Cover the answers. Say them out loud. If you can't answer one fluently, revisit that day before continuing.

Q01
A hospital database is leaked online. Nothing was altered. Which CIA element was broken?
Confidentiality — unauthorized people now have access to data they shouldn't see.
Q02
What is the only real difference between a white hat and a black hat hacker?
Written authorization. Same skillset — the signed scope of work is the legal line.
Q03
What does -rwxr-xr-- translate to in numeric permission notation?
754. Owner = 7 (rwx), Group = 5 (r-x), Others = 4 (r--).
Q04
TeamTNT renamed its malware to "kworker." How do you verify if a process is real?
ls -l /proc/[PID]/exe — reveals the actual binary path, which can't be spoofed like the name field.
Q05
What does a SYN flood attack actually do, step by step?
Sends thousands of SYN (step 1) requests without ever completing step 3 (ACK). Server holds resources open for each half-finished connection until exhausted and crashes — exactly what MafiaBoy did in 2000.
Q06
WannaCry exploited which protocol at which OSI layer? What two defenses were needed?
SMBv1 at Layer 7 (Application). Two defenses: block TCP port 445 at firewall (Layer 4) AND patch/disable SMBv1 (Layer 7). One alone wasn't sufficient.
Q07
How many usable host addresses does a /24 subnet provide? What's the broadcast address for 192.168.5.0/24?
254 usable hosts. Broadcast = 192.168.5.255.
Q08
The Baidu DNS hijack redirected 100M+ users without touching Baidu's servers. How?
DNS registrar records were poisoned — the "phone book entry" for baidu.com was changed to an attacker-controlled IP. Every DNS lookup returned the wrong address.
Q09
Why did the Target 2013 breach spread so far once attackers got inside?
Flat, unsegmented network — POS systems and vendor access zone were on the same network with no subnet boundary or firewall rules between them. One vendor credential = access to everything.
Q10
How did the PyPI supply chain attacks execute malicious code without the victim doing anything?
Malicious packages executed Python code on pip install — before the developer even imported the library. The attack was in the install command itself, not the code the developer wrote.
