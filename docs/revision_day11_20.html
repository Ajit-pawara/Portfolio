Day 11 — Windows Internals for Security

Core Concepts

  • NT Architecture: HAL → NTOSKRNL → subsystems (Win32, POSIX)
  • Security Account Manager (SAM): Stores local password hashes at C:\Windows\System32\config\SAM
  • LM & NTLM Hashes: LM is weak (split into 2×7-char halves), NTLM is MD4-based
  • LSASS: Local Security Authority Subsystem Service — manages authentication, stores credentials in memory
  • Event Logs: Security log (4624=logon, 4625=failed, 4688=process creation)
  • Registry: HKLM\SAM (hashes), HKLM\SYSTEM (boot key), HKCU\Software\Microsoft\Windows\CurrentVersion\Run (persistence)
🔴 Real Incident — NotPetya (2017): The world's costliest cyberattack leveraged Windows SMB vulnerabilities (EternalBlue). It propagated across networks by dumping LSASS credentials using Mimikatz, then using WMI and PsExec to move laterally. Total damage: >$10B.

Quiz: What registry hive contains the SYSTEM account's boot key, needed to decrypt SAM hashes?

Answer: HKLM\SYSTEM — Without the SYSTEM hive, SAM hashes are encrypted with the SysKey and cannot be extracted.

Day 16 — Cryptography I: Hashing vs Encryption

Core Concepts

  • Hashing: One-way, deterministic, fixed-size output. sha256sum file.txt
  • Encryption: Two-way — reversible with a key. openssl enc -aes-256-cbc -in plain.txt -out cipher.enc
  • Common Hash Algorithms: MD5 (128-bit, broken), SHA-1 (160-bit, deprecated), SHA-256 (256-bit, current standard)
  • Salting: Adding random data before hashing prevents rainbow table attacks
  • Integrity vs Confidentiality: Hashing verifies integrity (tamper detection), encryption provides confidentiality (secrecy)
🔴 Real Incident — LinkedIn Password Breach (2012): LinkedIn stored passwords as unsalted SHA-1 hashes. When 167M password hashes were leaked, most were cracked within days using rainbow tables and GPU clusters because SHA-1 was fast and there was no salt.

Quiz: Why is SHA-1 considered deprecated for password storage?

Answer: SHA-1 is too fast (GPU can compute billions per second) and has known collision attacks (SHAttered, 2017). Modern systems use bcrypt, scrypt, or Argon2 — deliberately slow, memory-hard algorithms.

Day 17 — Cryptography II: Symmetric vs Asymmetric

Comparison

  • Symmetric (AES): One key for encrypt and decrypt. Fast. Used for bulk data. Key exchange problem.
  • Asymmetric (RSA, ECC): Public key encrypts, private key decrypts. Slow. Used for key exchange and digital signatures.
  • TLS Handshake: ClientHello → ServerHello + Certificate → Key Exchange (RSA/ECDHE) → symmetric session key → encrypted channel
  • Perfect Forward Secrecy (PFS): Ephemeral Diffie-Hellman ensures past sessions can't be decrypted even if the private key is compromised
🔴 Real Incident — Heartbleed (CVE-2014-0160): A buffer over-read in OpenSSL's Heartbeat extension leaked up to 64KB of server memory per request, including private RSA keys. Attackers could decrypt previously captured TLS traffic. 500K+ servers affected.

Quiz: How does TLS use both asymmetric and symmetric cryptography in a single session?

Answer: Asymmetric (RSA/ECDHE) is used during the handshake to securely exchange a symmetric session key. Symmetric (AES) then encrypts all subsequent data for performance.

Day 18 — Cryptography III: SSL/TLS Certificates & PKI

Core Concepts

  • X.509 Certificate: Public key + identity + signature + validity period
  • Certificate Chain: Root CA → Intermediate CA → Server/Leaf certificate
  • Certificate Revocation: CRL (Certificate Revocation List) or OCSP (Online Certificate Status Protocol)
  • Self-Signed CA Lab: openssl req -x509 -newkey rsa:4096 -keyout ca.key -out ca.crt -days 365
  • PKI Trust Model: Browsers ship with ~150 trusted root CA certificates. Any can sign for any domain.
🔴 Real Incident — DigiNotar CA Breach (2011): A Dutch Certificate Authority was compromised by attackers who issued 531 fraudulent certificates for domains including google.com, torproject.org, and CIA.gov. Iranian users were MITM'd for months using a fake Google certificate signed by the compromised CA.

Day 19 — Authentication vs Authorization

Core Concepts

  • Authentication (AuthN): "Who are you?" — verifying identity via passwords, biometrics, MFA, certificates
  • Authorization (AuthZ): "What are you allowed to do?" — permissions, roles, ACLs, policies
  • IDOR: Insecure Direct Object Reference — a classic authZ failure where a user can access another user's data by changing an ID parameter
  • Session Management: Cookies, JWT tokens, session hijacking, fixation attacks
  • Broken Access Control: #1 on OWASP Top 10 — occurs when AuthN is checked but AuthZ is not
🔴 Real Incident — Capital One Data Breach (2019): A former AWS employee exploited a misconfigured WAF that allowed SSRF to the AWS metadata service. The real vulnerability was authorization — the IAM role attached to the WAF had access to S3 buckets containing 100M+ credit applications. Authentication was fine, but authorization was broken.

Quiz: What is the difference between horizontal and vertical privilege escalation?

Answer: Horizontal = accessing another user's data at the same privilege level (e.g., user A reads user B's inbox). Vertical = gaining higher privileges (e.g., user becomes admin).

Day 20 — Web Recon: Directory Busting & Technology Fingerprinting

Core Techniques

  • Directory Busting: Brute-forcing hidden paths — admin panels, backups, config files, dev endpoints
  • gobuster: gobuster dir -u https://target.com -w /usr/share/wordlists/dirb/common.txt -x php,txt,zip
  • ffuf: ffuf -u https://target.com/FUZZ -w wordlist.txt — faster, more flexible than gobuster
  • WhatWeb: whatweb https://target.com — passive technology fingerprinting
  • Wappalyzer: Browser extension — identifies JS frameworks, CMS, analytics, CDN, server software

Common Discovery Targets

  • /admin, /wp-admin, /backup, /config, /api, /swagger
  • /.git, /.env, robots.txt, sitemap.xml
  • /server-status, /phpinfo.php, /info.php
🔴 Real Incident — British Airways Magecart (2018): Magecart Group found an unsecured staging admin path on ba.com via directory busting. They injected a 23-line JavaScript skimmer into the payment page, capturing 380,000 payment card details over 2 weeks. BA was fined £20M under GDPR.

Quiz: What's the difference between gobuster's dir mode and vhost mode?

Answer: dir mode finds directories/files on a single domain. vhost mode discovers virtual hosts (subdomains) by sending different Host headers and monitoring response size differences.
