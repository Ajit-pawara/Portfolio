Subnetting & Routing Basics

Networks aren't one giant flat blob — they're divided into controlled segments. Today you understand how that division works, why the Target breach happened because of it, and how to use it in labs and interviews.

THEORY

What is subnetting? Every IP address has two parts: a network part (which network you're on) and a host part (which device you are). Subnetting controls exactly where that dividing line sits — splitting one big network into smaller, isolated sub-networks.

Plain English: think of an IP like a postal address — "Building 192.168.1, Apartment 45." The subnet decides how many apartments are in the same building, and whether your mailman needs to visit a different post office (router) to reach a different building.

The subnet mask tells your device which bits are "network" and which are "host." A mask of 255.255.255.0 = first 3 octets are network, last is host. In CIDR notation: /24 (first 24 bits locked as network).

Network address
192.168.1.0/24
Identifies the subnet — never assigned to a device
Broadcast address
192.168.1.255
Sends to ALL devices on subnet — never assigned either
Usable host range
.1 → .254
254 devices can live here
Default gateway
192.168.1.1
The router — traffic goes here when destination is outside this subnet

CIDR quick reference:

/24 = 255.255.255.0
254 hosts — most common in home/small networks
/16 = 255.255.0.0
65,534 hosts — large enterprise networks
/30 = 255.255.255.252
2 hosts only — point-to-point router links
/32
Single host — used in firewall rules for one exact IP

Routing: when your device sends data outside its subnet, it forwards the packet to the default gateway (router). The router checks its routing table and forwards toward the right network. Each hop repeats this until the packet arrives. That's it — routing is just a chain of forwarding decisions.

Security angle: subnet boundaries are firewall enforcement points. A compromised device in one subnet can't directly reach another subnet — traffic must cross a router where rules can block it. No segmentation = one breach can reach everything.
// real named incident
2013 — TARGET CORPORATION BREACH
Flat network + zero segmentation = 40M cards stolen
Attackers entered via a compromised HVAC vendor's credentials (Day 2's kill chain). Because Target's internal network had no subnet segmentation between vendor access zones and POS systems, attackers moved laterally straight to payment terminals and stole 40 million card numbers over weeks. Had POS systems been in a restricted subnet with inter-subnet firewall rules, the vendor credentials would have been worthless beyond the vendor zone. This is now the textbook case study for why subnet segmentation is non-negotiable in enterprise security — and the $18.5M settlement is the price of a flat network.
// attacker vs defender mindset
⚔ Attacker
  • "What other subnets can I reach from here?" — scanning different IP ranges after initial access is called network discovery.
  • A flat unsegmented network is a gift — one foothold reaches everything.
  • Routing tables and ARP caches reveal the network layout before moving laterally.
🛡 Defender
  • "Are critical systems in their own subnet with least-privilege firewall rules?"
  • Proper segmentation limits blast radius — one compromised subnet doesn't mean all are.
  • Monitor inter-subnet traffic — unexpected lateral movement is one of the clearest signs of active intrusion.

INTERVIEW READY
Q: "Why is network segmentation important from a security perspective?"
A: "Segmentation limits blast radius. The 2013 Target breach is the clearest real example — attackers entered through a third-party HVAC vendor's credentials, but because POS systems weren't segmented into a restricted subnet, they moved laterally across a flat network and stole 40 million card numbers. Proper subnetting with inter-subnet firewall rules enforcing least-privilege routing would have contained the breach to the vendor zone. I think of each subnet boundary as a mandatory firewall enforcement point — the more boundaries between crown-jewel systems and everything else, the harder lateral movement becomes."
