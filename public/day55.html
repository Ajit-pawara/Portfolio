INTERVIEW READY
Q: "Walk me through a typical penetration test from start to finish."
A: "A pentest has five phases. 1) Reconnaissance: gather information using OSINT (theHarvester, Shodan), passive DNS, and social engineering. 2) Scanning: Nmap to discover live hosts and open ports, vulnerability scanning with Nuclei or Nessus. 3) Exploitation: Metasploit for known exploits, manual web attacks (SQLi, XSS) through Burp Suite. 4) Post-exploitation: privilege escalation, lateral movement, persistence, data exfiltration. 5) Reporting: document findings with severity ratings, proof-of-concept evidence, and remediation recommendations. Each phase feeds into the next."
Q: "What is the most important thing you learned from the Phase 3 labs?"
A: "That manual exploitation gives you understanding, but automation gives you speed. You need both. Knowing how to manually craft a SQL injection payload teaches you how the database processes queries, which helps you write better detection rules later. But in a real engagement, you'd use sqlmap after confirming the injection manually. The balance between manual depth and automated breadth is what separates script kiddies from professional pentesters."
Q: "How do you prepare for a certification like OSCP or PNPT based on your Phase 3 experience?"
A: "The Phase 3 curriculum directly maps to OSCP objectives: Nmap scanning, web vulnerability assessment, exploitation with Metasploit and manual techniques, Linux and Windows privilege escalation, password cracking, and pivoting. For OSCP specifically, focus on manual exploitation without Metasploit's auto-pwn modules. For PNPT, practice active directory attacks, OSINT, and the full pentest report format. The THM/HTB methodology from Phase 3 directly translates to exam machine methodology."
