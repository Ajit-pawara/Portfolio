Quiz — can you answer these without looking back?

Q1: What is the difference between a SIEM and an EDR?
SIEM aggregates logs from many sources for centralized monitoring and alerting. EDR runs on endpoints to detect, investigate, and respond to threats at the host level. They're complementary — EDR feeds data into the SIEM.
Q2: Name all 7 phases of the incident response lifecycle.
1. Preparation, 2. Detection & Analysis, 3. Containment, 4. Eradication, 5. Recovery, 6. Post-Incident Activity (Lessons Learned), 7. (NIST adds) Coordination.
Q3: What is the STIX/TAXII protocol used for?
STIX is a structured language for describing cyber threat intelligence (indicators, campaigns, threat actors). TAXII is the transport protocol for sharing STIX data between organizations and MISP instances.
Q4: What is the difference between qualitative and quantitative risk assessment?
Qualitative uses descriptive scales (High/Medium/Low) based on expert judgment. Quantitative uses numerical data (ALE = SLE * ARO) to calculate monetary risk. Quantitative is more precise but requires more data.
Q5: What is the "blast radius" concept in Zero Trust?
Blast radius is the extent of damage if a single user, device, or workload is compromised. Zero Trust limits blast radius through micro-segmentation, least privilege access, and continuous verification — no implicit trust anywhere.
Q6: What is the difference between SAST and DAST in DevSecOps?
SAST scans source code statically during development (white-box). DAST scans the running application dynamically (black-box). SAST finds code-level issues, DAST finds runtime configuration and logic issues.

2021 — COLONIAL PIPELINE RANSOMWARE — COMPLETE IR LIFECYCLE CASE STUDY
DarkSide ransomware shut down the largest fuel pipeline in the US; the IR response demonstrates every Phase 4 concept in a single event
The Colonial Pipeline incident is the perfect Phase 4 case study. Preparation: Colonial had some security controls but lacked network segmentation and offline backups. Detection: The breach was detected when a ransom note appeared — initial access via an expired VPN account with no MFA went unnoticed for over an hour. Containment: They shut down the entire pipeline proactively, preventing lateral movement to OT systems. Eradication: The DarkSide ransomware was removed from IT systems. Recovery: They restored from backups and restarted operations after 6 days. Lessons Learned: The incident led to mandatory pipeline security directives, MFA requirements, and a national conversation about critical infrastructure security. Colonial paid $4.4M ransom — the FBI later recovered $2.3M of it.

INTERVIEW READY
Q: "You're a SOC analyst and see 500 failed logon events for a single user in 3 minutes. What do you do?"
A: "First, verify the source IP — is it internal or external? Check if MFA is enabled for that account. Look at the user's recent activity — are they also seeing successful logons from the same source? Correlate with other alerts — does the same IP appear in IDS logs (port scans) or WAF logs (web attacks)? If the source is external and the account has no MFA, I'd recommend temporarily disabling the account, blocking the IP at the firewall, and escalating to the incident response team for investigation of potential credential stuffing or brute force."
Q: "Walk me through your approach to a security assessment."
A: "I follow the NIST risk management framework: identify assets and their value, identify threats and vulnerabilities (scanning, penetration testing, log review), assess the likelihood and impact of each risk, recommend controls (administrative, technical, physical), and monitor continuously. For a typical assessment, I'd start with a scope definition, run automated vulnerability scanning (Nessus/OpenVAS), review configurations (CIS benchmarks), interview stakeholders, and produce a risk register with prioritized findings."
Q: "What's your biggest security weakness and how are you addressing it?"
A: "This is the time to be honest — for example, 'My weakest area is cloud security architecture. I understand the concepts of Zero Trust and IAM, but I haven't hands-on configured AWS security groups or Azure Policy. I'm addressing this by building a cloud security lab using AWS Free Tier to practice S3 bucket policies, security group rules, and CloudTrail log analysis, and I plan to pursue the AWS Security Specialty certification after the Security+ exam.'"
