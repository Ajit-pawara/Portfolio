Security+ Domains 3 & 4 Deep Review

Domain 3 covers implementation. Domain 4 covers operations. Today you master secure protocols, identity management, incident response procedures, and forensic best practices.

THEORY

Domain 3 — Implementation (25%): The largest exam domain covers everything you actually configure in a security role. Topics include: secure protocols (TLS 1.3, SSH, IPsec, HTTPS, DNSSEC, SRTP), PKI and certificates (CA hierarchy, certificate types, CRL, OCSP, key management), identity and access management (AAA framework, MFA methods, SSO/SAML/OAuth, Kerberos, account management), host security (antimalware, host firewall, HIDS/HIPS, application whitelisting), network security controls (firewall types, proxy server, load balancer, web filter, VPN concentrator), mobile security (MDM/UEM, containerization, BYOD policies), and wireless security (WPA3, EAP, PSK vs Enterprise, captive portal).

TLS 1.3
Replaces TLS 1.2 with faster handshake (1-RTT vs 2-RTT), removes obsolete ciphers (RC4, CBC), enables 0-RTT for session resumption. Required for modern HTTPS.
PKI Hierarchy
Root CA (offline) → Intermediate CA (online) → End-entity certificates (server/client). CRL lists revoked certs, OCSP provides real-time revocation checks.
MFA Methods
SMS codes (SMS-based, vulnerable to SIM swapping) < Time-based OTP (TOTP, app-based) < Push notification < FIDO2/U2F hardware keys (phishing-resistant).
Wireless Security
WPA3 (SAE handshake, forward secrecy) > WPA2 (still common, KRACK vuln) > WEP (broken). Enterprise mode uses 802.1X + RADIUS for per-user auth.
VPN Protocols
IPsec (site-to-site, IKEv2), SSL/TLS VPN (client-to-site, OpenVPN/WireGuard), L2TP/IPsec (legacy).
IAM Concepts
AAA: Authentication (who you are), Authorization (what you can do), Accounting (what you did). SSO, SAML, OAuth, and OpenID Connect.

Domain 4 — Operations & Incident Response (16%): Covers the incident response lifecycle (preparation, detection, containment, eradication, recovery, post-incident), forensic procedures (order of volatility, chain of custody, forensic image vs logical copy), data sources (logs, PCAP, netflow, syslog, SNMP, SIEM), mitigation techniques (firewall rules, IPS signatures, blacklisting, quarantining), digital forensics (disk forensics, memory forensics, network forensics), and disaster recovery (RTO, RPO, backup types, hot/cold/warm sites).

Sample PBQ scenario — IR walkthrough:

Scenario: A web server is exhibiting unusual outbound traffic at 3 AM. Step 1 — Detection: IDS alerts on outbound connection to known C2 IP. Step 2 — Containment: Remove server from load balancer, block C2 IP at firewall. Step 3 — Analysis: Take memory dump (via LiME), capture disk image (dd/FTK Imager). Step 4 — Eradication: Check for persistence mechanisms (cron jobs, scheduled tasks, registry run keys). Remove malware. Step 5 — Recovery: Patch vulnerability, restore from clean backup, add WAF rule. Step 6 — Post-Incident: Document timeline, update playbook, implement additional monitoring.
Key exam tip: Domain 4 questions often ask "What should you do FIRST?" in an incident. The answer is almost always contain the damage before investigating. "Isolate the system" or "Remove from network" comes before "Run a forensic analysis" or "Identify the root cause."
// attacker vs defender mindset
⚔ Attacker
  • PKI weaknesses are valuable targets — attackers who get access to a CA's private key can issue valid certificates for any domain (see DigiNotar 2011).
  • Incident response timelines are the attacker's enemy — the longer dwell time, the more data an attacker can steal. Attackers clear logs, disable auditing, and use LOLBins to slow forensic analysis.
  • Order of volatility matters to attackers too — volatile data (RAM, network connections) disappears on reboot. Attackers who force a system shutdown destroy the most valuable forensic evidence.
🛡 Defender
  • Implement MFA everywhere — especially for VPN, email, and admin accounts. FIDO2 hardware keys are phishing-resistant and the strongest MFA option.
  • Follow the order of volatility during forensic collection: RAM > network connections > process list > temporary files > disk. Collect in this order to preserve the most volatile evidence first.
  • Maintain chain of custody documentation — every time evidence changes hands, record who, when, and why. Missing chain of custody can make evidence inadmissible in court.

2023 — SECURITY+ PBQ SIMULATION — REAL IR WALKTHROUGH SCENARIO
Domain 4 PBQ scenario: compromised web server requiring disable account → collect memory → preserve logs → check lateral movement
A realistic Security+ PBQ might present this scenario: A web server is connecting to a known C2 IP address on port 8080 every 5 minutes. The task requires ordering these IR steps correctly: 1) Isolate the server from the network (containment), 2) Capture a memory dump to identify running processes (preserve volatility), 3) Collect system logs and export them to secure storage (preserve evidence), 4) Check Event IDs 4624 and 4625 for unauthorized logons (lateral movement investigation), 5) Check scheduled tasks and startup programs for persistence mechanisms (eradication prep), 6) Scan other servers for the same IoCs (scope assessment). The most common mistake: running a full forensic analysis before containing the threat. Containment always comes first.
HANDS-ON LAB

Goal: Practice IR triage, configure PKI certificates, and take a Domain 3-4 focused practice test.

  1. Create a self-signed CA and issue a server certificate using OpenSSL: openssl req -x509 -newkey rsa:2048 -keyout ca.key -out ca.crt -days 365 then issue a server cert signed by your CA.
  2. Configure TLS on a local Apache/Nginx server using your self-signed cert. Verify the connection with curl -v https://localhost and openssl s_client -connect localhost:443.
  3. Simulate an incident: create a suspicious process (simple Python script that makes an outbound request), then practice IR triage — capture memory with sudo cat /proc/[pid]/maps and network connections with ss -tunap.
  4. Use Autopsy (sleuthkit.org) or a free forensic tool to create a disk image of a USB drive and analyze it for hidden files, deleted data, and file metadata.
  5. Take a Domain 3-focused practice test (min 25 questions) and a Domain 4-focused test — score each and review every wrong answer.
Deliverable: OpenSSL CA + signed certificate (screenshot), incident triage log showing captured evidence in order of volatility, and practice test scores for both domains with wrong-answer review notes.

INTERVIEW READY
Q: "You arrive at a suspected breach. Walk me through your response."
A: "First, I'd confirm the incident — verify the alert isn't a false positive. Then immediately contain: isolate the affected system from the network to prevent lateral movement and data exfiltration. Next, I'd preserve evidence in order of volatility — capture RAM, network connections, process list, then disk. I'd document every step with timestamps for chain of custody. Then determine the scope — check other systems for the same indicators. Eradicate by removing the threat and patching the vulnerability. Recover from verified clean backups. Finally, conduct a lessons-learned session and update our incident response playbook."
Q: "What's the difference between a hot site, warm site, and cold site?"
A: "Hot site is a fully operational duplicate — hardware, data, networking, and personnel are ready to go within hours. it's expensive but has the lowest RTO. Warm site has hardware and networking but no current data — you restore from backups before going live. Cold site has just the facility — power, cooling, and cabling — but no hardware, no data, nothing. You bring everything in after a disaster. Cold site has the longest RTO but lowest cost."
Q: "What is the order of volatility and why does it matter?"
A: "Order of volatility is the sequence in which digital evidence should be collected, from most volatile to least: RAM contents, network connections, running processes, temporary files, and finally disk storage. It matters because volatile data disappears when power is lost — if you shut down a system before capturing RAM, you lose evidence of running malware, network connections to C2 servers, and encryption keys in memory."

ENTREPRENEUR ANGLE

Incident response readiness is a gap most small businesses never address until they're breached. You can offer an "IR Readiness Assessment" — review their incident response plan (or lack thereof), test their backup and recovery procedures, assess their forensic evidence collection capability — for $2,000-5,000. Many compliance frameworks (PCI DSS, HIPAA, SOC 2) require documented IR procedures.

PKI management is another opportunity. Small organizations often set up self-signed certificates or expired certs because they don't understand certificate management. You can offer a "TLS/SSL Certificate Audit" — check all certificates for expiration, weak ciphers, and proper chain validation — for $300-800. It's a quick, high-value service that every web-facing business needs.
