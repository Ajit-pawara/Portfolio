Vulnerability Scanning with Nessus/OpenVAS

Manual scanning finds what you look for. Automated scanners find what you miss.

THEORY

Vulnerability scanning vs pentesting: A scanner automates comparison of configs against a CVE database. Broad, fast, but false positives. A pentest manually validates through exploitation. Deep, verified, but slow. Both essential: scans monthly, pentests annually.

OpenVAS / Greenbone -- the open-source standard:

# Install Greenbone sudo apt install openvas -y sudo gvm-setup # Initialize (20-30 min) sudo gvm-start # Web interface at :9392

Scan categories by CVSS: Critical (9.0-10.0), High (7.0-8.9), Medium (4.0-6.9), Low (0.1-3.9), Log (info). Each finding includes CVE ID, CVSS score, description, affected versions, remediation.

Key insight: Unauthenticated scans find network vulns. Authenticated scans (with SSH/Windows creds) find missing patches and local misconfigs -- 5-10x more findings. Prefer authenticated when possible.
// attacker vs defender mindset
ATTACKER
  • Use scanner output as intelligence. OpenVAS findings give prioritized exploitation plan -- start with Critical + known Metasploit modules.
  • Don't trust scanners blindly -- false positives and logic flaws missed. Manually verify every Critical and High.
  • Scan off-peak. Aggressive scans can crash older devices (printers, IoT, legacy Windows).
  • Chain scanner findings: Critical on web server = initial access, High on internal system = lateral movement.
DEFENDER
  • Run authenticated monthly scans as baseline. Report shows what an attacker with network access sees.
  • Prioritize: Critical in 48h, High in 2wk, Medium in 30d.
  • Your IDS should detect vuln scans. No scan traffic in logs = misconfigured IDS.
  • Use patch management (WSUS, SCCM, Ansible). Average CVE-to-patch gap is 100+ days.
// real named incident
2013 -- TARGET BREACH -- 40 MILLION CREDIT CARDS
Post-breach forensic scanners identified 50+ unpatched vulns on Target's POS systems.
Attackers gained access via an HVAC vendor's compromised credentials, moved laterally to POS network. Post-breach Nessus scans revealed 50+ unpatched vulns including those exploited. Kaptoxa malware scraped card data from POS memory. 40M credit cards + 70M customer records stolen. Target paid $18.5M in settlements and $200M+ total. Regular scanning would have identified the lateral movement path.
HANDS-ON LAB

Goal: Install: sudo apt install openvas -y && sudo gvm-setup (20-30 min)....

  1. Install: sudo apt install openvas -y && sudo gvm-setup (20-30 min).
  2. Start: sudo gvm-start. Access https://127.0.0.1:9392 (admin/password from setup).
  3. Create Target: Configurations > Targets > New Target. Enter an IP.
  4. Create Task: Scans > Tasks > New Task. Select 'Full and Fast', select target. Start.
  5. Monitor progress in Tasks view (15-45 min). Review findings as they appear.
  6. Analyze: click task > findings by severity. Open Critical finding: CVE, CVSS, description, remediation.
Deliverable: Vulnerability scan report with target info, config, findings by severity, and one Critical finding breakdown.

INTERVIEW READY
Q: "Vulnerability scan vs pentest?"
A: "Scan: automated CVE comparison, broad/fast/false positives. Pentest: manual exploitation, narrow/slow/verified. Scans monthly, pentests annually."
Q: "Authenticated vs unauthenticated scan?"
A: "Unauthenticated: network-facing only (banners, open ports). Authenticated: login + enumerate installed software, missing patches, local users. 5-10x more findings with auth."
Q: "Prioritize vulnerability remediation?"
A: "CVSS: Critical (9.0+) in 48h, High (7.0-8.9) in 2wk, Medium (4.0-6.9) in 30d. Also: internet-facing vs internal, exploit existence, asset criticality, compensating controls."
