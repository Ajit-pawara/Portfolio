Nmap Advanced Scripts & Evasion

Nmap's NSE turns a port scanner into a vulnerability assessment tool.

THEORY

Nmap Scripting Engine (NSE): 600+ Lua scripts organized by category: vuln, exploit, auth, discovery, brute, safe, intrusive. The most famous: smb-vuln-ms17-010 (EternalBlue).

Evasion techniques:

# Decoy scan -- hide IP among decoys sudo nmap -sS -D RND:10 10.10.10.1 # Fragment scan -- split probe packets sudo nmap -sS -f 10.10.10.1 # Timing control -- slow down to blend sudo nmap -sS -T1 10.10.10.1 # Source port manipulation -- DNS port 53 often trusted sudo nmap -sS --source-port 53 10.10.10.1

Combine: sudo nmap -sV -sC target runs safe defaults. nmap --script=vuln target checks 50+ CVEs.

Key insight: Evasion trades stealth for speed. Decoys = 10x traffic. -T1 = 30x slower. Use minimum evasion needed.
// attacker vs defender mindset
ATTACKER
  • Run NSE vuln scripts as first pass after version discovery. Output often tells you the CVE + Metasploit module.
  • Don't assume evasion by default. Try standard scan first. Escalate to fragments if you see filtered ports.
  • Customize NSE with --script-args for target-specific fingerprints.
  • Log every scan with timestamps. Intrusive scripts can crash services.
DEFENDER
  • NSE also for defenders: nmap --script=vuln your-server finds vulns before attackers do.
  • Monitor for fragment scans -- fragmented packets that don't match normal traffic = Nmap evasion.
  • Watch for decoy patterns: multiple IPs scanning identically in short window = Nmap decoys.
  • Block unused ports at firewall, not just host. NSE on 445 can't exploit SMB if firewall blocks 445.

2017 -- WANNACRY/ETERNALBLUE -- 200,000+ SYSTEMS
Nmap's NSE smb-vuln-ms17-010 script allowed defenders to rapidly scan for EternalBlue.
WannaCry infected 200K+ systems across 150 countries using EternalBlue (CVE-2017-0144), an SMBv1 vulnerability. Defenders used NSE script smb-vuln-ms17-010 -- a single command could scan entire networks for unpatched systems in seconds. The NHS used it to identify affected systems. The script became the standard triage tool during the outbreak.

INTERVIEW READY
Q: "What is NSE in a pentest?"
A: "NSE is Nmap's Lua scripting engine with 600+ scripts. I use discovery (smb-enum-shares, http-enum), vuln (smb-vuln-ms17-010), auth (ftp-anon). -sC = safe defaults; --script=vuln = CVEs. I write custom scripts for specific targets."
Q: "Evasion techniques to bypass firewalls?"
A: "Decoy (-D), fragment (-f/--mtu), timing (-T1, --scan-delay, --max-rate), source port (--source-port 53), data length (--data-length). Minimum evasion for the target -- evasion costs time."
Q: "Safe vs intrusive NSE?"
A: "Safe: never crash target -- smb-enum-shares, http-title, ssl-cert. Intrusive: may disrupt -- smb-vuln-ms17-010 (can crash Samba), http-sql-injection, brute-force. Safe in production; intrusive with written auth only."
