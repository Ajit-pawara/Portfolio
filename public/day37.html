Passive Reconnaissance & OSINT

The best attacks start before you touch a single system. Today you gather intelligence from public sources.

THEORY

Passive reconnaissance (OSINT): Gathering information without directly interacting with the target. Sources include DNS records, search engines, social media, job postings, leaked credentials, CT logs, Shodan, Censys. The target cannot detect passive recon because you never send a packet to their infrastructure.

# DNS recon whois target.com # Registrant info, nameservers dig target.com ANY @8.8.8.8 # All DNS record types dig target.com AXFR # Zone transfer (rare gold) # Google dorking site:target.com filetype:pdf # Exposed documents site:target.com intitle:"index of" # Directory listing # Shodan shodan search hostname:target.com # Internet-facing services

Certificate Transparency: Every SSL cert is logged in public CT logs. Query them to find subdomains not in DNS: curl -s 'https://crt.sh/?q=%25.target.com&output=json' | jq -r '.[].name_value' | sort -u. Often reveals 5-10x more subdomains than DNS brute forcing.

Key insight: Passive recon is always the first step. The more you learn without touching the target, the less noise you make. The quieter you are, the longer you remain undetected during active ops.
// attacker vs defender mindset
ATTACKER
  • Start every engagement with full OSINT before any Nmap. Look for admin panels, employee emails, tech stacks, leaked creds.
  • Google dork: site:target.com filetype:sql OR filetype:env OR filetype:bak -- finds misconfigured servers.
  • Search job postings -- security job descriptions list specific firewall brands and SIEM tools.
  • Check Pastebin, GitHub, GitLab for exposed code and credentials using truffleHog.
DEFENDER
  • Monitor CT logs for your domains. If a cert appears you didn't request, someone may be phishing your domain.
  • Reduce OSINT footprint: remove employee details from public pages, use generic job descriptions.
  • Search for your own company on Shodan, Censys. Protect what you find before attackers find it.
  • Implement DNS zone transfer ACL -- only authorized secondaries should get zone transfers.
// real named incident
ONGOING -- SHODAN EXPOSED ICS DEVICES
Researchers use Shodan to discover thousands of unauthenticated industrial control systems.
Security researchers routinely use Shodan.io to find internet-connected ICS devices. Queries for port 502 (Modbus), 1911 (Niagara Fox), 44818 (Ethernet/IP) return thousands of exposed devices. In 2019, a researcher found 4,000+ Modbus devices accessible without authentication -- water treatment controllers, power grid monitors, hospital building automation. Many ran on default passwords like admin/admin. The same method for finding web servers finds water pumps and electrical substations, discoverable by any teenager with basic Shodan queries.
HANDS-ON LAB

Goal: Install tools: sudo apt install whois dnsutils jq -y && go install -v github.com...

  1. Install tools: sudo apt install whois dnsutils jq -y && go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest.
  2. WHOIS: whois example.com. Record registrant, dates, nameservers, admin email.
  3. DNS: dig example.com ANY @8.8.8.8 and dig example.com MX. Record A, MX, NS, TXT.
  4. CT logs: curl -s 'https://crt.sh/?q=%25.example.com&output=json' | jq -r '.[].name_value' | sort -u
  5. Google dork: site:example.com filetype:pdf OR filetype:docx in browser. Document findings.
  6. Shodan: shodan search hostname:example.com (requires free API key). Record ports and services.
Deliverable: OSINT profile with DNS records, subdomains, CT log findings, Google dork results, Shodan data.

INTERVIEW READY
Q: "What is OSINT and why first step?"
A: "OSINT gathers info from public sources without touching the target -- undetectable. It reduces active recon time and often reveals exposed creds and internal hostnames that make exploitation trivial."
Q: "Certificate Transparency logs for recon?"
A: "CT logs record every SSL cert issued, including all SANs (domain + subdomains). Query at crt.sh: curl + %25.target.com pattern. Reveals staging, dev, internal subdomains -- often 5-10x more than DNS brute forcing."
Q: "Passive vs active recon?"
A: "Passive does not interact with target -- whois, DNS to public resolvers, CT logs, Shodan, Google dorking. Cannot be detected. Active sends packets -- Nmap, directory busting, service probing. Can be detected. Passive first, active only after exhausting passive sources."

ENTREPRENEUR ANGLE

OSINT services are a standalone business. Companies want to know their public exposure before mergers, after breaches, for compliance. An 'External OSINT Assessment' ($2k-5k) delivers a report of public domains, exposed creds, sensitive documents. Free tools (Shodan, crt.sh, theHarvester) produce a report that helps reduce external attack surface. Many companies run this quarterly.
