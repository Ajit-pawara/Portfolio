Wireless Security Basics

Wi-Fi signals don't stop at walls. Today you understand WPA2/WPA3 authentication, capture handshakes, and learn how open networks leak everything.

THEORY

Wireless authentication — WPA2 vs WPA3: WPA2 (Wi-Fi Protected Access 2) uses the 4-way handshake for authentication. The client proves knowledge of the Pre-Shared Key (PSK) without revealing it directly. The handshake produces a Pairwise Transient Key (PTK) used for encrypting all subsequent traffic with AES-CCMP. WPA3 (2018) replaces the 4-way handshake with Simultaneous Authentication of Equals (SAE), which provides forward secrecy and is resistant to offline dictionary attacks that plague WPA2.

The WPA2 4-Way Handshake:

AP (Access Point) Client | | | ←——— EAPOL-Key (Anonce) ——————— | (1) AP sends its random number | | | ———→ EAPOL-Key (Snonce + MIC) ———→ | (2) Client sends its random + MIC | | | ←——— EAPOL-Key (MIC + GTK) —————— | (3) AP sends MIC + Group Temporal Key | | | ———→ EAPOL-Key (ACK) ———————————→ | (4) Client acknowledges | | ↑ If you capture these 4 packets, you can crack the PSK offline.

WPA2 PSK cracking — the math:

# PSK = PBKDF2(HMAC-SHA1, passphrase, SSID, 4096, 256) # PTK = PRF-X(PMK, "Pairwise key expansion", Min(AP_MAC,Client_MAC) || # Max(AP_MAC,Client_MAC) || Min(Anonce,Snonce) || Max(Anonce,Snonce)) # The MIC in message 3 is a hash of the PTK. When you guess the passphrase, # you derive the PTK and check if the MIC matches. If yes → correct password. # aircrack-ng automates this entire derivation & verification: aircrack-ng -w rockyou.txt -b AP_MAC handshake.cap
WPA3 fixes the offline dictionary attack: WPA2's PSK can be cracked offline if you capture the 4-way handshake. WPA3's SAE (Simultaneous Authentication of Equals) uses a Password Authenticated Key Exchange (PAKE) called Dragonfly. An attacker who captures a WPA3 handshake cannot perform offline brute force because each guess requires interacting with the real access point — the attack becomes online, which is slow, noisy, and detectable.
// attacker vs defender mindset
⚔ Attacker
  • Enable monitor mode on your wireless adapter: airmon-ng start wlan0. List nearby APs: airodump-ng wlan0mon.
  • Capture the 4-way handshake: airodump-ng -c CHANNEL --bssid AP_MAC -w capture wlan0mon. If no client is connecting, deauthenticate one: aireplay-ng -0 2 -a AP_MAC -c CLIENT_MAC wlan0mon.
  • Crack the handshake offline: aircrack-ng -w wordlist.txt -b AP_MAC capture-01.cap. Use rockyou.txt or generate targeted wordlists.
  • Set up a rogue Evil Twin AP with a similar SSID — if clients auto-connect, you can capture their handshake without knowing the password (PMKID attack).
🛡 Defender
  • Use WPA3 if all devices support it — SAE prevents offline dictionary attacks against captured handshakes.
  • Use a strong, random WPA2 passphrase — 16+ characters with mixed case, numbers, and symbols. Avoid dictionary words, names, and common phrases.
  • Disable WPS (Wi-Fi Protected Setup) — the WPS PIN brute force attack (Pixie Dust) can recover the PSK in seconds regardless of passphrase strength.
  • Implement 802.1X / RADIUS (WPA2-Enterprise) with individual credentials instead of a shared PSK. EAP-TLS with client certificates is the gold standard.
// real named incident
2017 — KRACK WPA2 ATTACK — CVE-2017-13077 THROUGH 13088
Protocol-level vulnerability in WPA2 allowed decryption of encrypted Wi-Fi traffic
Researcher Mathy Vanhoef discovered a fundamental weakness in the WPA2 protocol itself — not a brute force or passphrase crack, but a cryptographic protocol vulnerability. The KRACK (Key Reinstallation Attack) exploited the 4-way handshake's third message: an attacker could force the victim to reinstall an already-in-use encryption key by replaying the message. Each reinstallation reset the nonce and the replay counter used by AES-CCMP, allowing the attacker to decrypt packets, forge packets, and in some cases inject ransomware into HTTP connections. The attack worked against WPA2 with any passphrase strength — even a perfectly strong password was useless. The fix required patching all Wi-Fi client devices (phones, laptops, IoT), and the slow patch rollout left billions of devices vulnerable for years.
HANDS-ON LAB

Goal: capture a WPA2 4-way handshake, crack the PSK offline, and analyze wireless security settings.

  1. Enable monitor mode: sudo airmon-ng start wlan0. Verify with iwconfig — the interface should show "Mode:Monitor".
  2. Scan for nearby networks: sudo airodump-ng wlan0mon. Note the BSSID, channel, encryption type (WPA2 vs WPA3), and signal strength.
  3. Capture traffic on a specific AP: sudo airodump-ng -c CHANNEL --bssid AP_BSSID -w handshake wlan0mon.
  4. Force a client to reconnect (deauth attack): sudo aireplay-ng -0 2 -a AP_BSSID -c CLIENT_BSSID wlan0mon. Wait for the handshake in airodump-ng output.
  5. Verify the handshake was captured: aircrack-ng handshake-01.cap — it will confirm the number of handshakes found.
  6. Crack the PSK offline: sudo aircrack-ng -w /usr/share/wordlists/rockyou.txt -b AP_BSSID handshake-01.cap. Use a known password for testing.
Deliverable: A captured WPA2 4-way handshake file (.cap) and a screenshot of aircrack-ng successfully cracking the PSK (against your own AP only).

INTERVIEW READY
Q: "How does WPA2's 4-way handshake work and what's its weakness?"
A: "The 4-way handshake authenticates a client to an AP by proving knowledge of the PSK without transmitting it. The AP sends an Anonce, the client responds with a Snonce and a MIC (Message Integrity Code). The AP sends its MIC and the GTK. The client acknowledges. Both sides derive the same PTK from the PSK and nonces. The weakness: if an attacker captures all four messages, they can perform an offline dictionary attack to recover the PSK by trying candidate passwords and checking if the derived MIC matches."
Q: "What is the KRACK attack and why was it significant?"
A: "KRACK (Key Reinstallation Attack) exploited the third message of the WPA2 4-way handshake. An attacker within Wi-Fi range could replay the third message, causing the victim to reinstall an already-in-use encryption key. This reset the nonce and replay counter, allowing packet decryption and injection. Importantly, KRACK didn't require knowing the Wi-Fi password — it exploited a protocol flaw, not a weak password. Every WPA2-secured network was vulnerable. The fix required client-side patches to track whether keys had already been installed."
Q: "What are the security differences between WPA2-PSK and WPA2-Enterprise?"
A: "WPA2-PSK uses a single shared passphrase for all users — if the passphrase is compromised, every device on the network is vulnerable. WPA2-Enterprise uses 802.1X authentication with a RADIUS server, giving each user unique credentials. Enterprise also supports EAP-TLS with client certificates, which provides mutual authentication and prevents rogue AP attacks. The trade-off is complexity: Enterprise requires a RADIUS server and per-device configuration, while PSK just needs a password."

ENTREPRENEUR ANGLE

Wireless security assessments are a standard component of physical penetration testing and red teaming. Every office, warehouse, and retail location has Wi-Fi — and most are configured with minimal security. A single weak WPA2 passphrase or an enabled WPS PIN can compromise an entire corporate network.

Productized opportunity: "Wi-Fi Security Audit" — a $1,000-2,500 service where you physically visit a client's office, perform a wireless site survey (rogue AP detection, signal leakage analysis), test WPA2 passphrase strength, check for WPS vulnerabilities, and evaluate WPA2-Enterprise RADIUS configuration. Deliver a report with a "war driving" map showing signal leakage beyond the building perimeter, passphrase strength assessment, and remediation recommendations. Every law firm, healthcare provider, and financial services office should have this done annually.
