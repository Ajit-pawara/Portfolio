Password Cracking & Hashcat

Password strength is measured not in entropy, but in the time it takes to crack.

THEORY

Hashcat architecture: GPU-accelerated password recovery. Attack modes: 0=Dictionary, 1=Combination, 3=Brute-force (mask), 6=Hybrid dict+mask, 7=Hybrid mask+dict. The GPU parallelizes millions of hashes/sec vs CPU's thousands.

hashcat -m 1000 -a 0 hashes.txt rockyou.txt -m 1000 = NTLM | -a 0 = Dictionary hashcat -m 1000 -a 3 hashes.txt ?u?l?l?l?d?d?d?d Mask: Upper+3lower+4digits (Summer2024)

Hash types by frequency: NTLM (-m 1000, Windows), NetNTLMv2 (-m 5600, SMB), Kerberos TGS (-m 13100), SHA-512 (-m 1700, Linux shadow), bcrypt (-m 3200, modern apps), MD5 (-m 0, legacy).

Key insight: Passwords are the #1 attack vector. Hashcat shows you which of your org's passwords would survive a skilled attacker with a GPU cluster for 24 hours. Most won't.
// attacker vs defender mindset
ATTACKER
  • Start with wordlist + ruleset (best success/time ratio). rockyou.txt + best64.rule catches 50-60% of weak passwords.
  • Rule analysis: $1$2 (append year), s@a (leet), c (capitalize), so0 (leetspeak). Rules transform dictionary words.
  • Mask attacks for unknown patterns: ?u?l?l?l?d?d?d?d?d?s (Capital+3lower+5digit+special). Increment mode (--increment) to try all lengths.
  • Loopback mode: use --loopback to feed cracked passwords back through rules. Many use same word with different capitalizations.
DEFENDER
  • Set minimum password length (14+ chars). MFA everywhere. Password manager for all accounts.
  • Use bcrypt (cost 12+) or Argon2id. NTLM is unavoidable (Windows), but force LAPS for local admin passwords.
  • Audit passwords: hashcat against your own hashes. No passwords cracking in 24h on a single GPU = acceptable.
  • Training matters: 'Summer2024!' cracks in 2 min with rules. 'GT7k*eJz2!pL' takes millennia.
// real named incident
2009 -- ROCKYOU BREACH -- 32 MILLION PASSWORDS
RockYou stored passwords in cleartext. Released password list still used today for cracking.
RockYou (social gaming company) had a SQL injection vulnerability exposing 32M accounts. Passwords stored in cleartext. The rockyou.txt wordlist (14GB compressed) was extracted from this breach and remains the standard pentesting wordlist 15+ years later. Passwords analyzed: most common '123456', average password length 8 characters, 65% reused on other services.
HANDS-ON LAB

Goal: Extract NTLM hashes from Windows (with permission): secretsdump.py or Meterprete...

  1. Extract NTLM hashes from Windows (with permission): secretsdump.py or Meterpreter hashdump.
  2. Format: user:RID:LM:NTLM:::. Keep only NTLM portion for hashcat.
  3. Run dictionary attack: hashcat -m 1000 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt --show shows cracked.
  4. Add rules: hashcat -m 1000 -a 0 hashes.txt rockyou.txt -r /usr/share/hashcat/rules/best64.rule
  5. Try mask attack: hashcat -m 1000 -a 3 hashes.txt ?u?l?l?l?d?d?d?d?d --increment --increment-min 6
  6. Loopback on cracked: hashcat -m 1000 --loopback cracked.txt -r /usr/share/hashcat/rules/best64.rule
Deliverable: ['Hashcat session output: Speed (H/s), Progress, Time remaining, Cracking results with user:password.', 'Show system: GPU model, drivers, hashcat version.', 'GUI showing hashcat launched in terminal with attack modes.']
# Dictionary attack (fastest, best start) hashcat -m 1000 -a 0 ntlm.txt /usr/share/wordlists/rockyou.txt # Dictionary + rules (50-60% success) hashcat -m 1000 -a 0 ntlm.txt rockyou.txt -r best64.rule # Mask attack (pattern guessing) hashcat -m 1000 -a 3 ntlm.txt ?u?l?l?l?d?d? -i --i-min 6 # Show cracked passwords hashcat -m 1000 --show ntlm.txt # Loopback mode (cracked -> rules again) hashcat -m 1000 --loopback ntlm.txt -r best64.rule
INTERVIEW READY
Q: "Hash mode selection?"
A: "NTLM=1000, NetNTLMv2=5600, Kerberos TGS=13100, SHA-512=1700, bcrypt=3200, MD5=0, SHA1=100. Match hashcat's -m number. Use hashid tool if unsure."
Q: "Dictionary v mask v brute-force attacks?"
A: "Dictionary: wordlist, fast, effective. Mask: position-specific patterns (?u?l?d), no wordlist. Brute: all combos of charset, slowest. Dictionary+rules best ratio of time:success (50-60% in seconds). Mask for unknown patterns. Pure brute last resort."
Q: "Defend against hashcat?"
A: "Password length >14 chars. bcrypt/Argon2id. Enforce password manager. LAPS for local admin. MFA everywhere. Audit with hashcat first."
