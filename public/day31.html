Malware Types Virus, Worm, Trojan, Ransomware

Malware is not one thing. Each type spreads, hides, and executes differently. Today you catalog every major malware family and how they operate.

THEORY

Classifying malware by behavior: Malware is categorized by how it infects, how it spreads, what it does, and how it hides. Understanding these categories is essential for both defense (choosing the right detection tool) and analysis (knowing what to look for in a sample).

Major malware families:

Virus
Attaches to a legitimate file or program. Requires human action to spread (opening a file, running a program). Infects hosts one by one. Classic: CIH (Chernobyl), Melissa, ILOVEYOU.
Worm
Self-propagating. Spreads automatically across networks without human action. Exploits network services and vulnerabilities. Classic: Morris, Conficker, WannaCry, NotPetya.
Trojan
Disguised as legitimate software. Doesn't self-replicate — relies on social engineering to trick users into installing. Creates backdoors, steals data, or drops other malware. Classic: Zeus, Emotet, NanoCore RAT.
Ransomware
Encrypts victim's files and demands payment for the decryption key. Modern variants also exfiltrate data (double extortion). Classic: CryptoLocker, Ryuk, REvil, LockBit.
Rootkit
Designed to hide its presence and other malware from detection tools. Operates at kernel level (ring 0) or below (bootkits, hypervisor). Extremely difficult to detect without offline analysis.

YARA rule for malware classification:

# YARA — pattern-matching tool for malware identification rule NotPetya_Ransomware { strings: $mbr = { 33 C0 8E D0 BC 00 7C 8B F4 50 07 50 1F FB FC BF 00 06 } $msg = "CRITICAL_PROCESS_DIED" wide ascii condition: $mbr at 0 or $msg } # Run: yara notpetya.yara suspect-file.exe
The blur line: Modern malware rarely fits one category. Emotet started as a banking trojan, evolved into a botnet, and then became a dropper for ransomware. WannaCry is a worm (self-propagating via EternalBlue) that delivered ransomware. Classification helps with analysis but don't expect clean boundaries in real samples.
// attacker vs defender mindset
⚔ Attacker
  • Use trojans for initial access — craft a phishing email with a malicious Office document or ISO file that drops a RAT (Remote Access Trojan).
  • Deploy ransomware as the payload of last resort — encrypt everything, demand payment. Double extortion (exfiltrate + encrypt) increases pressure.
  • Use rootkits to maintain persistence — hide processes, files, registry keys, and network connections from EDR tools.
  • Develop polymorphic malware that changes its signature on each infection to evade signature-based AV detection.
🛡 Defender
  • Use behavior-based detection (EDR) rather than signature-only AV — EDR detects process injection, unusual file encryption, and C2 beaconing regardless of hash.
  • Implement application whitelisting — only allow known, approved executables. Blocks trojans and unknown malware by default.
  • Run YARA rules on critical systems — custom rules detect specific malware families based on strings, patterns, and behavior.
  • Maintain offline backups with immutable copies — the only reliable defense against ransomware. Test the restoration process regularly.
// real named incident
2017 — NOTPETYA RANSOMWARE — $10 BILLION IN DAMAGES
A Russian military cyberweapon disguised as ransomware destroyed systems globally
NotPetya was a wiper disguised as ransomware — it permanently destroyed the Master Boot Record (MBR) and encrypted files with a key that couldn't be recovered even if the ransom was paid. It spread using the EternalBlue exploit (leaked NSA tool) to propagate across networks without user interaction. The worm infected Maersk (shipping giant lost $300M in a week), Merck (pharma, $870M loss), FedEx ($400M loss), and countless other organizations. Total damages exceeded $10 billion, making it the most destructive cyberattack in history. Though initially attributed to the Russian military as an attack on Ukraine, it spread globally and caused collateral damage on an unprecedented scale.
HANDS-ON LAB

Goal: classify malware families, write YARA rules, and perform static analysis on a sample (in sandbox).

  1. Download a known malware sample from a malware repository (TheZoo, VirusShare) into an isolated VM or sandbox. Never run on your host.
  2. Use file and strings to identify the file type and extract readable strings: file suspect.exe && strings suspect.exe | head -50.
  3. Check VirusTotal hash (if sample is known) to see AV detection rates and classification labels.
  4. Write a YARA rule to detect the sample based on unique strings or byte patterns. Use yara rule.yara suspect.exe to test.
  5. Use pestudio or pecheck to analyze the PE header: imported functions, suspicious sections, timestamps, and compilation artifacts.
  6. Classify the sample: is it a virus, worm, trojan, or ransomware? Justify based on behavior and characteristics.
Deliverable: A written malware analysis report with file type, strings, YARA rule, PE analysis, and classification with justification.
# Step 1 — Basic file identification (in sandbox VM) file suspect.exe PE32 executable (GUI) Intel 80386, for MS Windows # Step 2 — Extract readable strings strings suspect.exe | grep -iE 'http|https|cmd|run|encrypt|decrypt' http://malicious-c2.example.com/gate.php cmd.exe /c del %s # Step 3 — Check VirusTotal (use the SHA256 hash) sha256sum suspect.exe # Submit hash to virustotal.com to see detection rates # Step 4 — Write a YARA rule cat > detect.yara << 'EOF' rule SuspectSample { strings: $c2 = "malicious-c2.example.com" $cmd = "cmd.exe /c del" condition: $c2 or $cmd } EOF yara detect.yara suspect.exe SuspectSample suspect.exe <-- matched! # Step 5 — PE analysis with pestudio or pecheck pecheck suspect.exe | grep -i 'suspicious\|import\|section' Sections: .text .rdata .data .rsrc <-- unusual section names Imports: CreateProcess, WriteProcessMemory, VirtualAllocEx <-- classic process injection imports
INTERVIEW READY
Q: "What's the difference between a virus and a worm?"
A: "A virus attaches itself to a legitimate file or program and requires user action (opening a file, running a program) to spread. A worm is self-propagating — it spreads automatically across networks by exploiting vulnerabilities or using network protocols, without requiring any user interaction. Worms spread much faster and wider than viruses. NotPetya and WannaCry were worms that delivered ransomware payloads."
Q: "How does a trojan differ from a virus?"
A: "A trojan masquerades as legitimate software — the user voluntarily installs it thinking it's useful. It doesn't self-replicate like a virus or worm. Trojans are commonly delivered through phishing emails, fake download pages, or pirated software. Once installed, they can steal data, create backdoors, or act as a downloader for other malware like ransomware."
Q: "What is a rootkit and why is it dangerous?"
A: "A rootkit is designed to hide its presence (and other malware) from the operating system and security tools. It operates at kernel level, intercepting system calls and filtering out its own files, processes, registry keys, and network connections. Rootkits are extremely difficult to detect because the OS itself can't be trusted once a rootkit is installed. Detection usually requires booting from an external forensic environment and analyzing memory dumps or disk images offline."
AI TOOLS
YARA rule generator
Describe a malware family (strings, behaviors, IOC patterns) — an LLM can generate a YARA rule with appropriate strings, conditions, and metadata tags.
Malware behavior explainer
Paste a strings output or API call list into an LLM — ask what the malware does and how to classify it. Useful for quick triage of unknown samples.
Static analysis assistant
Ask for help interpreting PE header fields: "What does it mean if the compile timestamp is 0x0?" or "Why would packed sections have high entropy?"
VirusTotal API analyzer
Use an LLM to help interpret VirusTotal results — ask it to break down the classification tags from multiple AV vendors into a simple malware family name.
