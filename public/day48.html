Wireless Attacks Aircrack-ng

Wi-Fi security can be broken without touching the target device. Today you capture WPA2 handshakes, crack PSK passwords, and understand enterprise Wi-Fi risks.

THEORY

How WPA2 works and why it can be cracked: WPA2-PSK uses a Pre-Shared Key (the Wi-Fi password) to derive the Pairwise Master Key (PMK) through PBKDF2-SHA1 hashing. When a client connects, a 4-way handshake occurs between client and access point. This handshake contains enough information (the PMKID and the handshake messages) to verify password guesses offline. The attacker doesn't need to be connected to the network — they just need to capture one successful handshake. Even without a client connecting, the PMKID attack (introduced in 2018) can extract the hash directly from the AP's beacon frame without any client interaction.

The Aircrack-ng toolchain:

# 1. Enable monitor mode — puts the wireless card in RFMON mode airmon-ng start wlan0 airmon-ng check kill # 2. Discover nearby networks and clients airodump-ng wlan0mon # 3. Capture handshake from a specific BSSID + channel airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon # 4. Deauth a client to force reconnection (captures handshake) aireplay-ng -0 5 -a AA:BB:CC:DD:EE:FF -c CLIENT_MAC wlan0mon # 5. Crack the PSK from the captured handshake aircrack-ng -w /usr/share/wordlists/rockyou.txt capture-01.cap

The PMKID attack: In 2018, researcher Jens Steube discovered that many access points include the PMKID in the first EAPOL frame of the handshake. The PMKID is computed as HMAC-SHA1(PMK, "PMK Name" + MAC_AP + MAC_STA). This means the attacker can request the PMKID by sending an EAPOL frame, and the AP responds with the hash — no clients needed, no deauth required. Use hcxdumptool or bettercap for this attack.

Key insight: WPA2 cracking is a hash cracking problem, not a network attack problem. Once you have the handshake (.cap file), you convert it to Hashcat format (.hccapx or .22000) and crack it on a GPU. The capture is the hard part; the cracking follows Day 47's methodology exactly.
// attacker vs defender mindset
⚔ Attacker
  • Use a wireless adapter that supports monitor mode and packet injection (Alfa AWUS036ACH, TP-Link TL-WN722N) — internal laptop Wi-Fi cards usually don't support injection.
  • First try the PMKID attack with hcxdumptool — it's faster, quieter, and requires no deauth that might alert the target or IDS.
  • If PMKID fails, use airodump-ng to find a network with an active client. Deauth (-0 3) sends 3 deauth packets — enough to force a reconnection without being obvious.
  • Convert .cap to Hashcat format: cap2hccapx capture-01.cap capture.hccapx — then crack at GPU speed with mode 22000.
🛡 Defender
  • Use WPA3 if hardware supports it — WPA3 uses SAE (Simultaneous Authentication of Equals) which is resistant to offline dictionary attacks because the handshake doesn't leak verifiable hash material.
  • For WPA2 networks, use a 20+ character random PSK — "Summer2024!" will crack in minutes; "9aKp!7xR$2mWqZ*3bL#8" will not crack in centuries.
  • Enable 802.11w (Management Frame Protection) — this prevents deauth attacks by requiring encrypted management frames, making it impossible to force a client to reconnect.
  • Deploy WIDS/WIPS (Wireless Intrusion Detection/Prevention) — systems like Snort or Kismet detect deauth floods, PMKID probes, and monitor-mode anomalies.
// real named incident
2017 — KRACK ATTACK (CVE-2017-13077 THROUGH 13087) — WPA2 PROTOCOL
Researchers discovered a vulnerability in the WPA2 protocol allowing traffic decryption without the network password
Researcher Mathy Vanhoef discovered the KRACK (Key Reinstallation Attack) vulnerability — a fundamental flaw in the WPA2 protocol itself, not an implementation bug. The attack targeted the 4-way handshake's third step: an attacker can force nonce reuse by replaying handshake messages, causing the client to reinstall an already-in-use encryption key. This allowed attackers within Wi-Fi range to decrypt traffic, inject malware, and hijack TCP connections without knowing the network password. Every Wi-Fi device with WPA2 was affected — billions of devices globally. The discovery led to WPA3 development acceleration and mandatory security patches from Microsoft, Apple, Google, and Linux distributors. The CVSS score was 8.1, and the vulnerability class was protocol-level, meaning it couldn't be fixed by a single vendor — every device had to be patched individually.
HANDS-ON LAB

Goal: capture a WPA2 4-way handshake and crack the PSK password using Aircrack-ng and Hashcat.

  1. Check your wireless adapter: iwconfig. Enable monitor mode: airmon-ng start wlan0. Run airmon-ng check kill to free the adapter.
  2. Discover networks: airodump-ng wlan0mon. Note a target BSSID, channel, and verify at least one client is associated.
  3. Capture traffic on the target channel: airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w handshake wlan0mon. Leave this running.
  4. In a second terminal, deauth a connected client: aireplay-ng -0 3 -a AA:BB:CC:DD:EE:FF -c CLIENT_MAC wlan0mon. The client reconnects — the handshake is captured.
  5. Verify the handshake: aircrack-ng handshake-01.cap. It should show "1 handshake captured" with the target BSSID.
  6. Crack the PSK: aircrack-ng -w /usr/share/wordlists/rockyou.txt handshake-01.cap. If you don't have a target, use a test handshake from a publicly available sample (WPA2 Sec shared on GitHub).
Deliverable: A captured WPA2 handshake .cap file and the cracked PSK (if using a test network). Document every command used with the output for each step.

INTERVIEW READY
Q: "Explain how you would perform a WPA2 handshake capture and PSK crack in a penetration test."
A: "First I'd put my wireless adapter in monitor mode using airmon-ng and scan for targets with airodump-ng. I prefer the PMKID attack when possible because it doesn't require clients or deauth — I send a PMKID probe with hcxdumptool and the AP responds with the hash. If that fails, I target a network with an active client, capture traffic on that channel, and send deauth packets to force reconnection. Once I have the handshake, I convert it to Hashcat format using cap2hccapx and crack the PSK with mode 22000 on a GPU."
Q: "What's the difference between WPA2-PSK and WPA3, and why does WPA3 prevent your attack?"
A: "WPA2-PSK uses a Pre-Shared Key to derive encryption keys through a 4-way handshake. The handshake material can be captured and verified offline against password guesses. WPA3 uses SAE (Simultaneous Authentication of Equals) based on Dragonfly handshake — a zero-knowledge proof protocol. In WPA3, the handshake doesn't leak any information that can be used for offline verification. Even if you capture the full WPA3 handshake, you cannot verify password guesses against it."
Q: "What is the PMKID attack and why is it significant?"
A: "Discovered in 2018, the PMKID attack exploits the fact that many access points include the PMKID in the first EAPOL frame. The PMKID is computed as HMAC-SHA1(PMK, 'PMK Name' + AP_MAC + Client_MAC). By sending a single EAPOL frame to the AP, the attacker gets back the PMKID hash — no clients, no deauth, no noise. It's significant because it makes WPA2 cracking completely passive and undetectable for roughly 50% of access points that support this behavior."

ENTREPRENEUR ANGLE

Wireless security testing is a specialized niche with less competition than web application testing. Most small and medium businesses have no idea how easy it is to crack their Wi-Fi — and many use default passwords on APs. You can offer a "Wireless Security Audit" service: walk the perimeter with a backpack-mounted Pi and Alfa adapter, capture handshakes passively, return with a report showing which networks crack in under an hour. Charge $1,500-3,000 per site.

Upsell opportunity: most companies don't know about WPA3 or 802.11w. After the audit, offer a "Wireless Hardening Package" — firmware updates, PSK rotation, WPA3 migration planning, and rogue AP detection setup. This is $2,000-5,000 per deployment, especially valuable for healthcare, finance, and legal firms with compliance requirements.
