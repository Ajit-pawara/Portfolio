DAY 78 OF 90 — PHASE 5: CERTIFICATION & PORTFOLIO

Security+ Domains 1 & 2 Deep Review

Domain 1 covers threats and attacks. Domain 2 covers architecture and design. Today you deep-dive into indicators of compromise, threat actors, and secure network design.

THEORY

Domain 1 — Attacks, Threats & Vulnerabilities (24%): This is the largest domain on the exam. You need to understand malware types (virus, worm, Trojan, ransomware, rootkit, spyware, adware), social engineering (phishing, spear-phishing, whaling, vishing, smishing, tailgating, pretexting, quid pro quo), application attacks (SQLi, XSS, CSRF, buffer overflow, race conditions), network attacks (DDoS, man-in-the-middle, DNS poisoning, ARP spoofing), wireless attacks (evil twin, deauthentication, jamming, KRACK), cryptographic attacks (birthday, rainbow table, brute force, dictionary, downgrade), and threat actors (APT, hacktivist, insider threat, script kiddie, organized crime).

Malware Types
Virus (file infector), Worm (self-propagating), Trojan (disguised), Ransomware (encrypts + extorts), Rootkit (kernel-level stealth), Spyware (data theft), Adware (forced ads).
Social Engineering
Phishing (mass email), Spear-phishing (targeted), Whaling (CEO), Vishing (voice), Smishing (SMS), Tailgating (physical follow), Pretexting (false scenario), Baiting (USB drop).
Application Attacks
SQLi (database injection), XSS (script injection), CSRF (forged request), Buffer Overflow (memory corruption), Race Condition (timing), Directory Traversal (path escape).
Network Attacks
DDoS (traffic flood), MITM (interception), DNS Poisoning (cache corrupt), ARP Spoofing (MAC impersonation), On-Path Attack (inline interception), Replay (capture + resend).
Cryptographic Attacks
Brute Force (try all keys), Dictionary (common passwords), Rainbow Table (precomputed hashes), Birthday (hash collision), Downgrade (force weaker protocol), Known Plaintext.
Threat Actor Types
APT (state-sponsored), Hacktivist (political), Insider (employee), Script Kiddie (low-skill), Organized Crime (financial), Shadow IT (rogue solutions).

Domain 2 — Architecture & Design (21%): Covers defense-in-depth, secure network design (DMZ, VLAN segmentation, VPN, NAC), virtualization and cloud (hypervisor, containers, IaaS/PaaS/SaaS, cloud deployment models), embedded and IoT systems (SCADA, PLC, IoT), secure facility design (mantraps, locks, biometrics, CCTV), and fundamental security concepts (CIA triad, non-repudiation, AAA, least privilege, separation of duties, defense-in-depth).

Sample PBQ scenario — configure a DMZ:

Scenario: Design a three-tier network architecture - Internet → Firewall → DMZ (web servers) - DMZ → Firewall → Internal (application servers) - Internal → Firewall → Private (database servers) Rules: 1. Allow HTTP/HTTPS from Internet → DMZ web servers (80/443) 2. Allow traffic from DMZ → Internal app servers only (8080) 3. Allow traffic from Internal → Private database (3306/5432) 4. Block all traffic from Private → Internet (no outbound) 5. Allow SSH management access from internal admin VLAN
Key exam tip: Domain 1 and Domain 2 test heavily on scenario-based questions. You'll see "An employee receives an email from their CEO asking them to wire money. What type of attack is this?" (Whaling). Or "A company wants to ensure data remains accessible even during a natural disaster. What should they implement?" (Redundancy + offsite backups).
// attacker vs defender mindset
⚔ Attacker
  • Phishing is the #1 initial access vector — 91% of attacks start with email. Attackers research targets on LinkedIn to craft convincing spear-phishing and whaling emails.
  • DNS poisoning and evil twin attacks target the trust at the protocol level. Setting up a fake Wi-Fi access point with a captive portal can capture credentials from anyone who connects.
  • Buffer overflow and race condition exploits require deep technical knowledge — most attackers use automated tools (Metasploit, Canvas) rather than writing exploits from scratch.
🛡 Defender
  • Defense-in-depth means never relying on a single control. Firewalls, IDS/IPS, endpoint protection, access controls, and security awareness training must work together.
  • Network segmentation (DMZ, VLANs) limits blast radius — an attacker who compromises a web server in the DMZ shouldn't have direct access to the database server.
  • Security awareness training is the most effective control against social engineering. Simulated phishing campaigns reduce click rates from 30% to under 5% over time.
// real named incident
2023 — MGM RESORTS RANSOMWARE — SOCIAL ENGINEERING + RAGNAR LOCKER
Attackers used vishing (phone call) to trick the help desk into resetting an employee's MFA — classic social engineering from Domain 1
The MGM Resorts ransomware attack perfectly illustrates Domain 1 social engineering and Domain 2 architecture failures. The attackers called MGM's help desk, impersonated a senior employee, and convinced the help desk to reset the employee's MFA registration. Once they had access, they moved laterally across the network — including into cloud infrastructure (Azure). The ransomware deployment (Ragnar Locker variant) encrypted 100+ ESXi hypervisors, forcing MGM to shut down hotel operations, casino floors, and reservation systems for over a week. The cost: estimated $100M in lost revenue. Multiple Domain 2 controls failed: no separation of IT and OT networks, lack of proper access controls for help desk password resets, and MFA bypass procedures that relied solely on caller identity verification.
HANDS-ON LAB

Goal: Practice attack classification, configure a DMZ in a virtual environment, and take a Domain 1-2 focused practice test.

  1. Create attack flashcards: write 20 attack types on cards with name, description, and prevention on the back. Shuffle and test yourself until you get all 20 right.
  2. Configure a DMZ in VirtualBox: create 3 VMs (web server, app server, database). Use iptables on a Linux VM as the firewall — allow HTTP/HTTPS to web, block direct DB access from internet.
  3. Run a simulated phishing attack using Gophish or SET (Social Engineering Toolkit): sudo setoolkit → Social Engineering Attacks → Credential Harvester → Site Cloner.
  4. Take a Domain 1-focused practice test (minimum 25 questions) from examcompass.com — score yourself and review wrong answers.
  5. Map 5 attack types from the news to Security+ Domain 1 categories — for each, write which threat actor type, which attack technique, and which control would prevent it.
Deliverable: Attack classification cheat sheet with all 20 attack types organized by Security+ category, screenshots of your DMZ iptables rules, and your Domain 1 practice test score with 3 wrong-answer reviews.

INTERVIEW READY
Q: "What's the difference between a virus and a worm?"
A: "A virus requires human action to spread — it attaches to a file or program and needs someone to execute it. A worm is self-propagating — it exploits network vulnerabilities to spread without human interaction. Examples: viruses spread via email attachments, worms like Stuxnet and WannaCry spread automatically across networks. Viruses infect files, worms infect networks."
Q: "What is defense-in-depth and why is it important?"
A: "Defense-in-depth is a layered security approach where multiple controls protect the same asset. If one control fails, others still provide protection. It's like layers of an onion — the outer layer might be a firewall, then an IDS, then access controls, then encryption, then application security. No single control is perfect — defense-in-depth ensures you're protected even when one layer is breached. It's the fundamental principle behind Domain 2's architecture and design objectives."
Q: "What's the difference between an IPS and an IDS?"
A: "IDS (Intrusion Detection System) monitors traffic and alerts when suspicious patterns are detected — it's passive. IPS (Intrusion Prevention System) sits inline and actively blocks malicious traffic in real-time. IDS gives you a report after detection; IPS stops the attack immediately. IPS can have false positives that block legitimate traffic, which is why some organizations use IDS for monitoring and manual response."
