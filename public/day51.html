Intro to TryHackMe & HackTheBox

Capture The Flag platforms are where theory meets hands-on practice. Today you create accounts, understand room/box categories, and solve your first challenges.

THEORY

Why CTF platforms accelerate learning: Traditional cybersecurity education follows a curriculum — learn X, then learn Y, then maybe practice Z. CTF platforms invert this: here's a vulnerable machine, figure it out. This forces you to apply knowledge immediately, combine techniques across domains, and develop the research muscle (Googling error messages, reading docs, adapting public exploits). The 1M+ users on TryHackMe and 500K+ on HackTheBox aren't wrong — these platforms produce job-ready practitioners faster than most university programs.

Platform comparison:

TryHackMe (THM) — Guided learning paths with step-by-step rooms — Browser-based Kali (no local VM needed) — Best for: beginners, structured learning, SOC/purple team — Subscription: free ($0), premium ($10/mo) HackTheBox (HTB) — Open-ended vulnerable machines with no handholding — Requires VPN connection to HTB network — Best for: intermediate/advanced, red team, pentest practice — Subscription: free (limited), VIP ($14/mo), VIP+ ($20/mo)

Your first session roadmap: Create free accounts on both platforms. On THM, start with the "Complete Beginner" learning path — rooms like "Linux Fundamentals," "Nmap," and "Burp Suite" revisit material you already know but add the CTF twist. On HTB, start with "Starting Point" — a curated set of easy machines with walkthroughs. The goal of today is not to root a hard box; it's to understand the platform mechanics, the VPN/network setup, and the rhythm of enumeration → exploitation → flag submission.

Key insight: The most common mistake beginners make on CTF platforms is spending hours on a box without taking notes. Every command, every finding, every thought process should be logged. Not only does this help you reproduce the solution, but it's a record of your methodology that you can review and improve. After 90 days, your personal CTF notes will be your most valuable study resource.
// attacker vs defender mindset
⚔ Attacker
  • On THM, read the room description carefully — it often hints at the technique you need to use. The room is a learning exercise, not a blind pentest.
  • On HTB, the hardest part is the first 10 minutes: find the attack surface. Run nmap with default scripts (-sC) and version detection (-sV) on all ports — then focus on what's unusual.
  • Use the "try harder" mentality — when stuck, enumerate more. Don't look for writeups until you've run full port scans, directory brute-forcing, and vulnerability scanning.
  • Submit flags as you go — both platforms have user flags (low-priv) and root flags (admin). Get the user flag first, then escalate to root. Each flag is a milestone.
🛡 Defender
  • CTF platforms are also learning tools for defenders — THM's "blue team" path (SOC Level 1) teaches log analysis, incident response, and threat hunting.
  • Study the writeups for boxes you couldn't solve — even after you get the flag, reading how others approached it reveals techniques you hadn't considered.
  • Use the platform's discussion forums to ask specific questions. "I enumerated ports X Y Z, found service A, tried exploit B, got error C" — this forces structured thinking.
  • Track your solve times — as you improve, your time-to-root should decrease. This is a measurable metric of skill progression.

HANDS-ON LAB

Goal: create accounts on TryHackMe and HackTheBox, connect to their networks, and complete at least one introductory room/machine on each.

  1. Create a free account at tryhackme.com. On THM, start the "Linux Fundamentals" room — it has a browser-based Kali instance. Complete the tasks to get your first flag.
  2. On THM, use the AttackBox (browser-based Kali) or connect via OpenVPN. Complete the "Nmap" room — practice scanning the target VM with different flags.
  3. Create a free account at hackthebox.com. Go to "Starting Point" — download the VPN configuration file and connect: sudo openvpn starting_point.ovpn.
  4. In HTB Starting Point, select the "Meow" machine (easiest). Run ping 10.129.x.x to verify connectivity, then scan with nmap and find the open telnet port.
  5. Complete the "Meow" box: connect via telnet with default credentials (root:root), find the flag in the root directory, and submit it on the HTB platform.
  6. For each challenge, document in your daily log: target IP, open ports, services detected, exploitation steps, and flags obtained with timestamps.
Deliverable: Screenshots or notes showing: (1) Your THM profile with at least one completed room, (2) Your HTB Starting Point progress with at least one rooted machine, (3) A documented methodology for each challenge.

INTERVIEW READY
Q: "What's the difference between TryHackMe and HackTheBox, and which one would you recommend to a beginner?"
A: "TryHackMe is guided learning with step-by-step rooms, a browser-based attack box, and structured learning paths. It's ideal for beginners because you're never left entirely on your own — rooms tell you what to do and teach as you go. HackTheBox is open-ended — you get an IP address and must figure out everything yourself. I recommend starting with THM for structured learning, then transitioning to HTB once you're comfortable with enumeration methodology. Many professionals use both: THM for learning new techniques, HTB for testing their skills under pressure."
Q: "How would you approach a HackTheBox machine you've never seen before?"
A: "I follow a consistent methodology: (1) Full port scan with nmap -p- to find all open ports. (2) Service version detection with -sV on discovered ports. (3) If HTTP is open, directory brute-forcing with gobuster or ffuf. (4) Vulnerability scanning with nikto or searchsploit for the discovered services. (5) If I find an exploit, I test it against the target. (6) After initial access, I immediately run enumeration for privilege escalation. Each step is documented with timestamps so I can identify where I spent too much time."
Q: "How many HTB boxes have you solved? Walk me through one you're proud of."
A: "I've solved 15+ HTB machines and completed the Complete Beginner path on THM. One box I'm proud of is [name of box] — it required chaining a SQL injection in a web application with a sudo misconfiguration for privilege escalation. The SQL injection was blind boolean-based, so I had to write a Python script to automate the extraction. The privesc was a custom binary with SUID that I hadn't seen before — I had to analyze it with strings and understand its behavior before crafting the exploit."
AI TOOLS
Writeup analyzer
After solving a box, paste the writeup into an LLM and ask it to extract the key methodology steps as a checklist — reinforces learning from successful solutions.
Stuck-on-CTF assistant
Describe what you've enumerated (ports, services, findings) without asking for the direct solution — the LLM can suggest what you might have missed or what to try next.
Note-taking template generator
Ask for a Markdown template for CTF/HTB note-taking — organized sections for recon, exploitation, privesc, and flags. Use this template for every box going forward.
Box difficulty ranker
Ask which HTB or THM boxes cover specific techniques — "Which easy boxes involve XSS?" or "Which starting point boxes teach SQL injection?" — plan your learning path.
