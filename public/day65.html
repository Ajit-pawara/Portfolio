MITRE ATT&CK Framework

ATT&CK is the encyclopedia of adversarial behavior. Today you learn to navigate the matrix, map real attacks to techniques, and use ATT&CK for defensive gap analysis.

THEORY

What is MITRE ATT&CK? MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) is a globally-accessible knowledge base of adversary tactics and techniques based on real-world observations. It organizes attacker behavior into a matrix where columns represent tactical goals (Initial Access, Execution, Persistence, etc.) and rows represent the techniques used to achieve those goals. As of 2025, ATT&CK v14+ covers 14 tactics and over 200 techniques across Enterprise (Windows, macOS, Linux, Cloud), Mobile, and ICS environments.

Mapping an Attack Chain: The SolarWinds compromise demonstrates ATT&CK's value. The attack chain maps to the matrix: T1078 (Valid Accounts) for initial access via compromised credentials, T1195 (Supply Chain Compromise) as the primary vector through the Orion build system, T1059 (Command and Scripting Interpreter) for Teardrop payload execution, T1003 (OS Credential Dumping) for lateral movement, T1071 (Application Layer Protocol) for encrypted C2 via HTTPS, and T1048 (Exfiltration Over Alternative Protocol) for data theft. This mapping allows defenders to identify exactly which controls would have mitigated each step.

Defensive Gap Analysis: The true power of ATT&CK is identifying control gaps. By mapping your detection coverage against the matrix, you find tactics with no visibility. For example, most organizations have strong Initial Access and Execution detection but weak Persistence and Exfiltration coverage — meaning attackers who get past the perimeter have wide latitude. ATT&CK-driven gap analysis produces a prioritized roadmap: "We need to build detection for T1133 (External Remote Services) because we have no visibility into VPN-based persistence."

Why this is essential: Before ATT&CK, defenders described threats in inconsistent ways — "the attacker used a dropper" vs "the attacker executed a payload via PowerShell" might refer to the same technique. ATT&CK provides a shared language. When a threat intel report says "APT29 used T1133," every SOC analyst in the world knows exactly what detection and mitigation strategies to deploy.
// attacker vs defender mindset
ATTACKER
  • Study ATT&CK to identify which techniques have the lowest detection rates in target environments — T1098 (Account Manipulation) for persistence, T1036 (Masquerading) to blend in.
  • Avoid high-visibility techniques (T1059.001 PowerShell has extensive logging). Switch to T1059.003 (Windows Command Shell), T1059.005 (Visual Basic), or T1059.007 (JavaScript).
  • Chain techniques that span multiple tactics to overwhelm defenders — a Defender can block one technique but a sequence of 10 across 14 tactics has more points of failure.
  • Use ATT&CK-to-CVE mappings (e.g., T1190 → CVE-2021-44228 for Log4Shell) to find unpatched systems and weaponize exploits against known techniques.
DEFENDER
  • Use the ATT&CK Navigator to create heatmaps of detection coverage — color-code each technique as Green (detected), Yellow (partial), Red (no detection). Share with management to justify tooling investment.
  • Map every SIEM rule and EDR alert to an ATT&CK technique — if a rule can't be mapped, it's either detecting something irrelevant or the technique isn't in the matrix yet.
  • Prioritize detection for techniques used by threat actors targeting your industry — if you're in education, ransomware TTPs (T1486, T1490) matter more than state-sponsored techniques.
  • Use ATT&CK to test defenses with purple team exercises — pick a technique, simulate it, measure detection, improve coverage. Iterate weekly.
// real named incident
2020 — SOLARWINDS / SUNBURST — 18,000 ORGANIZATIONS
Mandiant used MITRE ATT&CK technique mapping to attribute the SolarWinds compromise to APT29
Mandiant's incident response team mapped over 15 distinct ATT&CK techniques from the SolarWinds compromise chain, including T1195 (Supply Chain Compromise), T1078 (Valid Accounts), T1553 (Subvert Trust Controls — the SUNBURST fake certificate), and T1564 (Hide Artifacts). The specific combination of techniques — particularly T1553.002 (Code Signing) with a stolen DigiCert certificate and T1564.001 (Hidden Files and Directories) using the legitimate Orion directory — showed tooling overlaps with previous APT29 campaigns. This ATT&CK-based attribution was critical because the malware was custom, the C2 was encrypted, and traditional IoC matching would have missed the connection entirely.
HANDS-ON LAB

Goal: use ATT&CK Navigator to create a coverage heatmap and map a real attack chain to the matrix.

  1. Open the MITRE ATT&CK Navigator at mitre-attack.github.io/attack-navigator in your browser.
  2. Select "Enterprise Layer" and choose the latest ATT&CK version. The full matrix loads — 14 tactics with techniques underneath.
  3. Create a coverage layer: click "Layer 1" → "Properties" → "Layer Name" → "Current Coverage." Assign colors: green (fully covered), yellow (partial), red (no detection). Mark 5 techniques from Initial Access as green (e.g., T1078, T1195, T1133, T1190, T1566).
  4. Create a second layer for the SolarWinds attack chain. Add techniques T1195, T1078, T1553, T1564, T1059, T1003, T1071, T1048.
  5. Perform a gap analysis by overlaying the two layers — techniques in red on the coverage layer that appear in the attack chain layer are critical gaps.
  6. Export both layers as JSON (Layer → Export) and save them to your portfolio directory.
Deliverable: two ATT&CK Navigator layer files (JSON) — one for your simulated detection coverage and one for the SolarWinds attack chain. Together they demonstrate your ability to use ATT&CK for defensive gap analysis, a skill directly requested in SOC analyst job descriptions.

INTERVIEW READY
Q: "How do you use MITRE ATT&CK in a SOC environment?"
A: "I use ATT&CK for three main purposes. First, mapping alerts: every SIEM rule and EDR alert is tagged with the corresponding ATT&CK technique, which lets us measure coverage against the matrix. Second, gap analysis: when we're considering a new detection tool, we overlay its claimed coverage against our existing coverage to identify gaps. Third, intelligence-driven defense: when a threat intel report describes an APT group's TTPs, I can quickly check which of those techniques our environment can detect. ATT&CK provides a common language between threat intel, detection engineering, and incident response."
Q: "How did ATT&CK help during the SolarWinds investigation?"
A: "ATT&CK allowed analysts to map the entire kill chain — from supply chain compromise (T1195) through C2 (T1071) to exfiltration (T1048) — using standardized identifiers. This made it possible to correlate the SolarWinds incident with prior APT29 activity because the technique combinations matched. Without ATT&CK, investigators would have described the attack differently and the connection between campaigns might not have been spotted."
Q: "What's the difference between a tactic, a technique, and a procedure in ATT&CK?"
A: "A tactic is the 'why' — the adversary's goal (e.g., Persistence, Credential Access). A technique is the 'how' — the general method to achieve the goal (e.g., T1133 — External Remote Services for persistence). A procedure is the specific implementation — the exact tool, command, or sequence used (e.g., 'An adversary established persistence by adding a VPN user account with a stolen password'). Tactics are the columns, techniques are the rows, and procedures are the real-world examples you find in threat reports."

Attack chain analyzer

Coverage gap explainer

Purple team technique simulator
