Active Recon Nmap Basics

Passive recon tells you what's possible. Active recon tells you what's actually there.

THEORY

Nmap -- the network mapper: The de facto standard for network discovery. Sends crafted packets and analyzes responses to determine live hosts, open ports, services, OS, firewall rules.

Essential scan types:

# TCP SYN Scan (default, root) -- "stealth" scan sudo nmap -sS 10.10.10.1 # TCP Connect Scan (non-root) -- completes handshake nmap -sT 10.10.10.1 # UDP Scan -- slow, essential for DNS(53), SNMP(161) sudo nmap -sU 10.10.10.1

Port states: Open (service accepting), Closed (reachable, no service), Filtered (firewall blocking), Open|Filtered (UDP).

Key insight: SYN scan is default because it's fast and less detectable. Always follow with -sV on open ports. Start with top 100 ports (-F), not all 65535.
// attacker vs defender mindset
ATTACKER
  • Start with ping sweep: nmap -sn 10.10.10.0/24. No reason to scan dead IPs.
  • Scan top 100 ports first (-F), then full (-p-) only on hosts with interesting services.
  • Always use version detection (-sV). Apache 2.4.49 = exploitable; 2.4.51 = not.
  • Use -oN/-oX/-oG for every scan. Scans you don't save didn't happen.
DEFENDER
  • Block ICMP echo at firewall perimeter. Won't stop Nmap but removes easiest discovery.
  • Monitor for port scans via IDS/IPS. Snort has pre-built Nmap detection rules.
  • Use port knocking or VPN for management services. Move SSH off port 22.
  • Run regular Nmap scans of your own network externally. Find what attackers find.
// real named incident
2016 -- MIRAI BOTNET -- 2.5 MILLION IoT DEVICES
Mirai used Nmap-style SYN scans on Telnet port 23 to discover vulnerable IoT devices.
Mirai sent SYN packets to random IPs on port 23 (Telnet). SYN/ACK response = open, then it tried 61 default password combos. Within hours of source code release, 600K+ IoT devices were compromised (IP cameras, DVRs, routers). The 1.2 Tbps DDoS against Dyn DNS took down Twitter, Netflix, Reddit, Spotify. Attack chain: SYN scan to find port 23, then default creds. No exploits, no zero-days.
HANDS-ON LAB

Goal: Identify a target: tryhackme.com or your own router with permission....

  1. Identify a target: tryhackme.com or your own router with permission.
  2. Host discovery: nmap -sn 10.10.10.0/24. Record live hosts.
  3. SYN scan top ports: sudo nmap -sS -F 10.10.10.1. Interpret states.
  4. Version detection: sudo nmap -sS -sV -T4 10.10.10.1. Document services.
  5. OS detection: sudo nmap -O 10.10.10.1. Note fingerprint and confidence.
  6. Compare: nmap -sT 10.10.10.1 (Connect) vs SYN output for differences.
Deliverable: Nmap scan report with live hosts, port states, service versions, OS guess, and commands used.

INTERVIEW READY
Q: "SYN scan vs Connect scan?"
A: "SYN (-sS) sends SYN; on SYN/ACK sends RST before completion. Faster, root-only, less detectable. Connect (-sT) uses connect() syscall, completes full handshake. Slower, no root needed, logged by apps. Always SYN when root available."
Q: "Nmap port states?"
A: "Open (service accepting), Closed (reachable no service), Filtered (firewall blocking), Unfiltered (reachable undetermined), Open|Filtered (UDP common)."
Q: "Scan without IDS detection?"
A: "SYN scan, decoy (-D RND:5), fragment (-f), slow timing (-T1), source port 53. Blend with normal traffic. Minimum evasion needed."
